# JDSnack Decision Store

이 폴더는 여러 작업에 반복해서 영향을 주는 기술·운영 결정을 짧은 ADR로 보관합니다.

## 기록 규칙

- 파일명은 `ADR-NNN-short-title.md` 형식을 사용합니다.
- Context, Decision, Consequences를 포함합니다.
- 활성 spec이나 운영 문서가 참조하는 결정은 삭제하지 말고 superseded 상태로 남깁니다.
- 구현이 바뀌면 관련 ADR과 `docs/ARCHITECTURE.md`를 함께 확인합니다.
