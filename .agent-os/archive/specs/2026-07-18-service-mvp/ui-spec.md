# Service MVP UI 계약

## 화면

### 인증 진입

- 비로그인 사용자는 공개 홈 화면과 상단 로그인 CTA를 먼저 본다.
- 비로그인 상태에서는 분석 입력·JD 수집·분석 실행 입력과 버튼을 사용할 수 없고, 보호 API 호출도 발생하지 않는다. 로그인 후 해당 기능을 활성화한다.
- 상단 로그인 CTA를 누르면 중앙 Google 로그인 모달을 열고, 닫으면 배경 화면으로 돌아간다.
- 로그인 시작·callback·실패 상태를 표시한다.
- provider token과 secret은 화면 상태·localStorage·번들에 저장하지 않는다.

### 분석 입력

- 기존 이력서 텍스트 입력과 PDF/DOCX 업로드 중 하나를 선택한다.
- 업로드 파일은 텍스트 추출 후 화면 상태와 분석 요청에만 사용하고 파일 보관 기능을 제공하지 않는다.
- JD는 직접 붙여넣기 또는 사람인 URL 입력을 지원한다.
- 제출 중에는 중복 제출을 막고, 검증 실패는 필드 가까이에 표시한다.

### 분석 결과

- 성공 이력은 진단 결과와 JD 매칭 결과를 표시한다.
- 실패 이력은 `FAILED`, 사용자 안내, 재시도 CTA를 표시한다.
- 입력 스냅샷과 JD 본문은 상세 화면에서 확인할 수 있다.

### 분석 이력

- 최신순 목록에 성공·실패 상태를 구분해 표시한다.
- 상세 진입·재시도·삭제를 제공한다.
- 재시도 완료 시 기존 상세를 덮지 않고 새 이력 상세로 이동한다.
- 삭제 전 확인을 받고, 삭제 완료 후 목록에서 제거한다. 복구 CTA는 제공하지 않는다.

T4에서 사이드바의 `lockedItems`를 수정할 때는 `분석 내역`을 해금하고, 기능 정의가 없는 목업 잔재인 `템플릿`과 `키워드 사전` 항목을 목록에서 제거한다. 두 항목을 잠금 상태로 남기거나 기능을 새로 정의하지 않는다.

## 상태

- 인증: `unauthenticated | authenticating | authenticated | error`
- 분석: `idle | submitting | running | succeeded | failed`
- 이력 조회: `loading | loaded | empty | error`
- 삭제: `confirming | deleting | deleted | error`

## 프론트 경계

- 컴포넌트는 `services/` 계층을 통해서만 보호 API를 호출한다.
- API response 타입은 `types/`에 두고, 서버 오류를 사용자 메시지로 변환하는 책임은 service/hook 경계에 둔다.
- 분석 입력 화면은 이력서 라이브러리, JD 동기화, 원본 파일 다운로드 기능을 렌더링하지 않는다.
