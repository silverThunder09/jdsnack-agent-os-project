# 새로운 분석 시작 페이지 재구성 UI 명세

## 전체 레이아웃 (데스크톱)

```text
┌───────────┬──────────────────────────────────────────────────────────┐
│ 사이드바  │  상단 바: 브랜드                              (도움말/알림) │
│ ▣ 홈      ├──────────────────────────────────────────────────────────┤
│ ▸ 모의면접 │  새로운 분석 시작                                          │
│ ───────── │  채용 공고와 이력서를 연결하여 합격 가능성을 높이세요.      │
│ 🔒 분석내역│ ┌── 좌: 입력 단계 ─────────────┐  ┌─ 우: 안내 레일 ───┐ │
│ 🔒 이력서  │ │ 1. 채용 공고(JD) 입력         │  │ 분석이 이렇게 진행 │ │
│ 🔒 템플릿  │ │   [JD 링크][JD 내용 붙여넣기] │  │  돼요 (4단계)      │ │
│ 🔒 키워드  │ │ 2. 이력서 업로드 (PDF/DOCX)   │  │ 정확도 팁         │ │
│ 🔒 첨삭    │ │ 3. 분석 옵션 선택 (체크박스4) │  │ 지원 가능한 형식  │ │
│ 🔒 요금제  │ │   [ 분석 시작하기 → ]         │  └───────────────────┘ │
│ ───────── │ └───────────────────────────────┘                        │
│ 프로필(정적)│                                                          │
└───────────┴──────────────────────────────────────────────────────────┘
```

## 사이드바

- 브랜드 + 메뉴: 홈, 분석 내역, 이력서 관리, 템플릿, 키워드 사전, 맞춤 첨삭, 요금제 + 모의 면접.
- 활성: `홈`, `모의 면접`. 잠금: 나머지(잠금 아이콘, `aria-disabled`, 이동 불가).
- 하단 프로필/플랜 영역은 정적 표시(이름·플랜·만료일·플랜 관리 버튼은 비동작).
- 좁은 폭: 접힘/토글.

## 본문: 새로운 분석 시작 (홈 `/`)

- 제목 "새로운 분석 시작" + 서브카피.
- 좌(입력)|우(안내 레일) 2단.

### 1. 채용 공고(JD) 입력
- 탭: `JD 링크` / `JD 내용 붙여넣기`.
- `JD 링크`: URL 입력 + 지원 사이트 안내. 수집은 `POST /api/jd/fetch`.
- `JD 내용 붙여넣기`: 본문 textarea + 글자 수(예: `0 / 30,000`).
- 두 입력은 하나의 `jdText`로 귀결.

### 2. 이력서 업로드
- 드래그앤드롭 영역 + `파일 선택` 버튼.
- 지원 형식 안내: PDF, DOCX(최대 용량 표기). 업로드 후 파일 칩(이름·크기·제거).

### 3. 분석 옵션 선택
- 체크박스 4개:
  - `JD 적합도`(추천) — 이력서와 JD 핵심 키워드·역량 적합도. **동작(매칭)**.
  - `ATS 분석`(추천) — 통과 가능성·포맷·키워드 최적화. **준비중**.
  - `문장 첨삭` — 문장 표현·가독성·문법·전문성. **준비중**.
  - `키워드 분석` — 주요 키워드 누락·활용도. **준비중**.
- CTA `분석 시작하기 →`. 하단 안전 안내("입력 정보는 안전하게 처리되며 분석 후 즉시 삭제됩니다").

### 우측 안내 레일
- "분석이 이렇게 진행돼요": JD 수집 → 이력서 파싱 → AI 종합 분석 → 결과 제공.
- "분석 정확도를 높이는 팁".
- "지원 가능한 파일 형식": PDF, DOCX + 용량.

## 분석 실행 → 결과 화면

- `분석 시작하기`: JD(링크/본문)·이력서 파일·옵션≥1 검증. 미충족 시 해당 입력에 에러, 입력 보존.
- `JD 적합도` 실행: `diagnose/file`로 본문 추출 → `match`로 매칭.
- 결과 화면(셸 내 뷰 전환):
  - 매칭 결과: 점수 요약 카드 + 강점/gap/제안 패널.
  - 준비중 옵션: "준비중" 패널.
  - 로딩/실패 상태 표시, 실패 시 입력 보존. `새 분석` 버튼으로 입력 화면 복귀.

## 모의 면접 (`/interview` 또는 뷰)

- 기존 흐름 유지: 직무(+선택 맥락) 입력 → 질문(카테고리·키포인트)·전략·요약.

## 스타일

- 디자인 랭귀지: 인디고/바이올렛 강조, 라이트 배경, 둥근 흰 카드+소프트 섀도, 단계 번호 배지, 체크박스 카드, 안내 레일 카드.
- `App.css`를 셸 그리드 + 단계 카드 + 안내 레일 기준으로 정리.

## 접근성

- 입력 label 연결, 탭/체크박스/버튼 role 유지.
- 오류는 `role="alert"` 등 즉시 인지 영역.
- 결과 생성 시 aria-live 알림. 잠금 메뉴 `aria-disabled`.

## 마이그레이션 메모

- 기존 `ResumeFileInput`/`StatusMessage`/`AppShell`과 훅(`useDiagnose`/`useMatchPreview`/`useInterviewPreview`), `services/api.ts`, `types/diagnosis.ts` 재사용.
- App.tsx의 결과 표시(`SummaryCard`/`AnalysisPanel`/`EmptyState`)는 결과 화면에 재사용.
- `App.tsx`/`App.css`를 단계형 입력 + 결과 뷰 전환 구조로 재작성.
