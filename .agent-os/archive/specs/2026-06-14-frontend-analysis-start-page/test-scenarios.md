# 새로운 분석 시작 페이지 재구성 테스트 시나리오

## `TC-01` 셸 레이아웃 렌더

- 대응 AC: `AC-01`
- 절차: 앱에 진입한다.
- 기대 결과: 사이드바·상단 바·본문(제목+좌 입력|우 안내 레일)이 한 화면에 보인다.

## `TC-02` 사이드바·프로필·모의 면접

- 대응 AC: `AC-02`
- 절차: 사이드바와 좌하단을 확인하고, 잠금 항목과 모의 면접을 각각 클릭한다.
- 기대 결과: 잠금 항목은 이동이 없고, 모의 면접은 이동한다. 프로필/플랜 영역은 정적으로 표시된다.

## `TC-03` 3단계 입력 동작

- 대응 AC: `AC-03`
- 절차: JD `링크`/`붙여넣기` 탭을 전환하고, 이력서 파일을 올린다.
- 기대 결과: 붙여넣기에 글자 수가 보이고, 업로드 파일 이름·크기와 제거가 동작한다.

## `TC-04` 분석 옵션 선택·준비중 표시

- 대응 AC: `AC-04`
- 절차: 옵션 4개를 확인하고 선택/해제한다.
- 기대 결과: `JD 적합도`는 실행 대상이 되고, ATS/문장 첨삭/키워드는 "준비중"으로 안내된다.

## `TC-05` 분석 시작 → 매칭 결과 화면

- 대응 AC: `AC-05`
- 절차: JD와 이력서를 입력하고 `JD 적합도`를 선택해 `분석 시작하기`를 누른다.
- 기대 결과: 결과 화면으로 전환되어 매칭 점수·요약·강점·gap·제안이 표시된다.

## `TC-06` 준비중 옵션 결과 패널

- 대응 AC: `AC-04`, `AC-05`
- 절차: ATS/문장 첨삭/키워드 중 하나를 선택해 분석을 실행한다.
- 기대 결과: 결과 화면에 해당 항목이 "준비중" 패널로 표시되고 외부 호출이 없다.

## `TC-07` 검증·에러 보존

- 대응 AC: `AC-06`
- 절차: JD나 이력서 없이 실행, 그리고 `ai-local`에서 키 누락/Gemini 실패 상황을 만든다.
- 기대 결과: 검증 에러가 기존과 동일하게 표시되고 입력이 보존된다. 키 누락은 설정 실패, Gemini 실패는 재시도 안내로 표시된다.

## `TC-08` 기존 테스트 갱신·통과

- 대응 AC: `AC-06`
- 절차: `cd frontend && npm run lint`, `npm test`, `npm run build`, `npm run test:e2e`를 실행한다.
- 기대 결과: lint·단위·빌드·e2e가 통과한다.

## `TC-09` 반응형·안내 레일

- 대응 AC: `AC-07`
- 절차: 데스크톱 폭과 좁은 폭에서 화면을 확인한다.
- 기대 결과: 우측 안내 레일이 보이고, 좁은 폭에서 세로 스택으로 전환된다.
