# JD 입력 MVP 개발 계획

> 목표: 이력서 다음 단계에서 JD를 어떻게 받을지 문서로 먼저 고정한다.

## Task 1: JD 입력 요구사항 정리

- JD 텍스트 필수 입력과 JD 링크 선택 입력을 문서로 고정한다.
- 링크 자동 수집은 범위 밖으로 명확히 뺀다.

## Task 2: 비교 API 요청 계약 초안

- `POST /api/match/preview` 요청 구조를 문서로 정의한다.
- `resumeSource` 의미를 텍스트 본문 기준으로 정리한다.

## Task 3: 화면 흐름 설계

- 이력서 단계 다음에 JD 입력 단계가 오는 흐름을 정의한다.
- 오류 메시지와 입력 상태를 분리한다.

## Task 4: 추적 매핑 고정

- `REQ -> AC -> TC -> API/UI` 연결을 만든다.
- 2차 MVP AI 계약과 섞이지 않도록 이번 단계 범위를 닫는다.
