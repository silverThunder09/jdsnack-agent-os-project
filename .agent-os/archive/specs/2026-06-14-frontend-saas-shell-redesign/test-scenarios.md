# 프론트 SaaS 셸 재설계 테스트 시나리오

## `TC-01` 셸 레이아웃 렌더

- 대응 AC: `AC-01`
- 절차:
  - 앱에 진입한다.
- 기대 결과:
  - 사이드바·상단 바·본문(히어로→입력→결과)이 한 화면에 보인다.
  - 브랜드가 JDSnack으로 표시된다.

## `TC-02` 사이드바 잠금·프로필 부재

- 대응 AC: `AC-02`
- 절차:
  - 사이드바와 화면 좌하단을 확인한다.
  - 잠금 메뉴 항목을 클릭한다.
- 기대 결과:
  - 전체 메뉴가 보이고 홈/모의 면접만 활성이다.
  - 잠금 항목은 이동이 일어나지 않는다.
  - 계정/프로필·플랜 영역이 없다.

## `TC-03` 통합 분석 실행·입력 보존

- 대응 AC: `AC-03`
- 절차:
  - 이력서와 JD를 입력하고 `분석 시작`을 누른다.
- 기대 결과:
  - 페이지 이동 없이 진단과 매칭이 함께 실행되어 결과가 채워진다.
  - 입력값이 보존되어 재실행할 수 있다.

## `TC-04` 결과 요약 카드 표시

- 대응 AC: `AC-04`
- 절차:
  - 분석 실행 후 요약 카드를 확인한다.
- 기대 결과:
  - 이력서 진단 점수와 JD 적합도 카드만 표시된다.
  - ATS·누락 키워드·첨삭 카드는 없다.

## `TC-05` 진단·매칭 상세 프리뷰

- 대응 AC: `AC-04`
- 절차:
  - 분석 실행 후 상세 프리뷰를 확인한다.
- 기대 결과:
  - 진단 강점/개선, 매칭 강점/gap/제안이 표시된다.
  - 결과 생성 전에는 빈/안내 상태가 보인다.

## `TC-06` 모의 면접 목적지 동작

- 대응 AC: `AC-05`
- 절차:
  - 모의 면접 목적지로 이동해 직무를 입력하고 질문을 생성한다.
- 기대 결과:
  - 질문 목록(카테고리·키포인트)·전략·요약이 표시된다.

## `TC-07` 검증·에러 보존

- 대응 AC: `AC-06`
- 절차:
  - 빈 이력서, 너무 짧은 이력서로 각각 실행한다.
  - `ai-local`에서 키 누락/Gemini 실패 상황을 만든다.
- 기대 결과:
  - 입력 검증 에러가 기존과 동일하게 표시되고 입력이 보존된다.
  - 키 누락은 설정 실패 안내, Gemini 실패는 재시도 안내로 표시된다.

## `TC-08` 기존 테스트 갱신·통과

- 대응 AC: `AC-06`
- 절차:
  - `cd frontend && npm run lint`, `npm test`(vitest), `npm run build`, `npm run test:e2e`를 실행한다.
  - 셸/목적지 구조에 맞게 단위/e2e selector·흐름을 갱신한다.
- 기대 결과:
  - lint·단위 테스트·빌드·e2e가 통과한다.

## `TC-09` 반응형 전환

- 대응 AC: `AC-07`
- 절차:
  - 데스크톱 폭과 좁은 폭에서 화면을 확인한다.
- 기대 결과:
  - 데스크톱은 넓은 레이아웃, 좁은 폭은 세로 스택으로 전환되고 사이드바가 접힘/축소된다.
