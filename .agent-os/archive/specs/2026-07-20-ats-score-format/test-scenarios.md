# ATS 점수·포맷 진단 테스트 시나리오

## API

- **TC-01 유효 입력 성공**: 인증된 사용자가 50자 이상 이력서·JD를 보내면 ATS 응답의 점수와 필수 목록을 확인한다.
- **TC-02 결정론성**: 같은 요청을 두 번 실행해 응답의 점수·요약·목록이 같은지 확인한다.
- **TC-03 입력 검증**: 빈 값, 50자 미만, 10,000자 초과 이력서·JD가 기존 공통 오류 코드로 거절되는지 확인한다.
- **TC-04 인증 경계**: 세션 없는 ATS 요청이 401로 차단되는지 확인한다.

## UI

- **TC-05 옵션 해금**: ATS 카드에 `준비중`이 없고 활성 checkbox로 선택되는지 확인한다.
- **TC-06 결과 표시**: ATS 선택 후 ATS 점수, 포맷 점검, 키워드·제안 패널이 표시되는지 확인한다.
- **TC-07 선택적 호출**: ATS 미선택 시 `/api/ats/preview`가 호출되지 않고 기존 결과가 유지되는지 확인한다.
- **TC-08 내보내기**: ATS 성공 결과를 Markdown으로 내보낼 때 ATS 점수와 진단 항목이 포함되는지 확인한다.

## 회귀

- **TC-09 기존 분석 회귀**: JD 매칭·문장 첨삭·키워드 분석 선택과 기존 API 호출이 유지되는지 확인한다.
- **TC-10 정적 품질 게이트**: 백엔드 테스트, 프론트 lint/test/build/E2E, Docs Harness, Docker health/smoke가 통과하는지 확인한다.
