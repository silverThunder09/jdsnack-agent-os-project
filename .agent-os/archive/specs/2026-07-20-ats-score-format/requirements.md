# ATS 점수·포맷 진단 요구사항

## 목표

로그인한 사용자가 기존 이력서와 JD를 재사용해 ATS 관점의 정량 점수와 보완 항목을 확인할 수 있게 한다. 이번 첫 버전은 외부 AI 호출 없이 서버의 결정론적 휴리스틱으로 동일 입력에 동일 결과를 반환한다.

## 범위

### 포함

- 기존 분석 입력에서 ATS 분석 옵션 해금
- 인증된 `POST /api/ats/preview` 보호 API
- 이력서·JD 길이 검증
- ATS 점수(0~100), 포맷 점검, 키워드 매칭·누락, 개선 제안 반환
- 결과 화면·Markdown 내보내기 반영
- fixture/stub와 무관하게 실행되는 무료 정적 검증 경계

### 제외

- PDF 시각 레이아웃을 직접 판정하는 렌더링 엔진
- ATS 제품별 비공개 규칙 추정
- 외부 AI 호출·API key 추가
- 분석 이력 DB 스키마 확장과 과거 이력 재계산
- 키워드 분석·문장 첨삭·JD 매칭의 동작 변경

## 요구사항

### REQ-01 ATS 입력 재사용

인증된 사용자는 기존 분석 화면에서 이력서 추출 텍스트와 JD 본문을 ATS 진단에 사용할 수 있어야 한다.

### REQ-02 ATS 보호 API

서버는 `POST /api/ats/preview`에서 이력서와 JD를 검증하고 ATS 결과 DTO를 반환해야 한다. 인증되지 않은 요청은 공통 인증 경계에서 차단한다.

### REQ-03 결정론적 점수

동일한 이력서·JD 입력은 동일한 점수와 결과를 반환해야 하며 점수는 0 이상 100 이하이어야 한다. 점수는 추출 가능한 텍스트의 섹션 구조, JD 키워드 포함, 성과 수치, 연락처 단서의 신호를 조합한다.

### REQ-04 실행 가능한 결과

응답은 점수, 요약, 강점, 위험·누락, 개선 제안, 매칭 키워드, 누락 키워드, 포맷 점검을 포함해야 한다. 시각적 PDF 레이아웃을 확인하지 못한 경우 이를 과장하지 않는다.

### REQ-05 기존 흐름 보존

ATS를 선택하지 않은 분석의 기존 JD 매칭·문장 첨삭 흐름은 유지되어야 한다. ATS 실패는 다른 선택 결과를 숨기지 않는다.

### REQ-06 비밀정보 경계

ATS 구현은 브라우저에 secret을 요구하지 않으며, 외부 서비스와 통신하지 않는다.
