# 분석 결과 리포트 내보내기 수용 기준

- **AC-01** 로그인한 사용자가 성공(`SUCCEEDED`) 이력 상세에서 내보내기 진입점을 사용하면 Markdown 파일이 내려받아진다. (REQ-01)
- **AC-02** 본인 소유가 아닌 이력에는 상세 화면 자체가 열리지 않으므로 내보내기 진입점도 노출되지 않는다. 이력 상세 조회는 기존 `GET /api/analysis-histories/{historyId}` 소유권 경계를 그대로 사용한다. (REQ-02)
- **AC-03** 내보내기 파일은 이력에 저장된 `diagnosis`·`match` 결과만 섹션으로 포함하고, 저장되지 않은 옵션 섹션은 생성되지 않는다. (REQ-03)
- **AC-04** 내보내기 파일명에 이력 식별자와 생성일이 포함되고, 파일의 content type은 `text/markdown`이다. (REQ-04)
- **AC-05** `RUNNING` 또는 `FAILED` 이력에서는 내보내기 진입점이 비활성이거나, 실행 시 결과 없음을 명확히 안내하고 빈 성공 리포트를 만들지 않는다. (REQ-05)
- **AC-06** 내보내기 로직은 외부 API 호출이나 브라우저 secret 입력 없이 동작한다. (REQ-06)

- 문서·코드·테스트·CI 결과가 일치합니다.
