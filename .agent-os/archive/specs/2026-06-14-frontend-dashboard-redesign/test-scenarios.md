# 프론트 대시보드 전면 재설계 테스트 시나리오

## `TC-01` 입력·결과 동시 표시

- 대응 AC: `AC-01`
- 절차:
  - 앱에 진입한다.
- 기대 결과:
  - 입력 영역과 결과 영역이 한 화면에 함께 보인다.
  - 단계 번호로 이동하는 위저드 UI가 없다.

## `TC-02` 이력서 진단 패널 표시

- 대응 AC: `AC-02`, `AC-03`
- 절차:
  - 이력서를 입력하고 진단을 실행한다.
- 기대 결과:
  - 페이지 이동 없이 진단 패널에 점수·요약·강점·개선이 표시된다.
  - 입력한 이력서가 보존된다.

## `TC-03` JD 매칭 패널 표시

- 대응 AC: `AC-02`, `AC-03`
- 절차:
  - 이력서와 JD를 입력하고 매칭을 실행한다.
- 기대 결과:
  - 매칭 패널에 매칭 점수·요약·강점·gap·제안이 표시된다.

## `TC-04` 모의 면접 질문 패널 표시

- 대응 AC: `AC-02`, `AC-03`
- 절차:
  - 이력서(선택적으로 직무/JD)로 면접 질문 생성을 실행한다.
- 기대 결과:
  - 면접 패널에 질문 목록(카테고리·키포인트)·전략·요약이 표시된다.

## `TC-05` 군더더기 제거 확인

- 대응 AC: `AC-04`
- 절차:
  - 화면 상단과 헤더를 확인한다.
- 기대 결과:
  - 프로필/계정 버튼이 없다.
  - `STEP n / 3` 라벨이 중복 표시되지 않는다.

## `TC-06` 반응형 전환

- 대응 AC: `AC-06`
- 절차:
  - 데스크톱 폭과 좁은 폭에서 화면을 확인한다.
- 기대 결과:
  - 데스크톱은 입력|결과 2단, 좁은 폭은 세로 스택으로 표시된다.

## `TC-07` 검증·에러 동작 유지

- 대응 AC: `AC-05`
- 절차:
  - 빈 이력서, 너무 짧은 이력서로 각각 실행한다.
  - `ai-local`에서 키 누락/Gemini 실패 상황을 만든다.
- 기대 결과:
  - 입력 검증 에러가 기존과 동일하게 표시되고 입력이 보존된다.
  - 키 누락은 설정 실패 안내, Gemini 실패는 재시도 안내로 표시된다.

## `TC-08` 기존 테스트 갱신·통과

- 대응 AC: `AC-01`, `AC-05`
- 절차:
  - `cd frontend && npm run lint`, `npm test`(vitest), `npm run build`를 실행한다.
  - 대시보드 구조에 맞게 단위/e2e 테스트의 selector·흐름을 갱신한다.
- 기대 결과:
  - lint·단위 테스트·빌드가 통과한다.
  - e2e가 위저드 단계 전환 가정 없이 대시보드 흐름으로 통과한다.
