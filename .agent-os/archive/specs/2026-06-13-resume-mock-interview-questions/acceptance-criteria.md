# 모의 면접 질문 생성 수용 기준

## `AC-01` 면접 질문 생성 성공 흐름

- 사용자가 준비된 이력서로 면접 질문 생성을 요청하면 `POST /api/interview/preview`가 호출된다.
- 성공 응답에는 예상 면접 질문 목록과 준비 요약이 포함된다.
- 각 질문은 카테고리와 답변 키포인트를 함께 보여준다.

## `AC-02` 직무·JD 맥락 반영

- `jobTitle` 또는 `jdText`를 함께 제출하면 질문이 해당 직무/공고 맥락에 맞춰진다.
- 두 값이 모두 없으면 이력서만으로 일반 예상 질문이 생성된다.
- 맥락 값은 선택값이며 누락돼도 요청은 정상 처리된다.

## `AC-03` 운영 모드별 동작

- `ai-local` 모드에서는 실제 Gemini 응답으로 질문이 생성된다.
- `fixture` 모드에서는 외부 호출 없이 정적 샘플 질문이 반환된다.
- `stub` 모드에서는 준비중 안내가 반환되고 외부 호출이 발생하지 않는다.

## `AC-04` 입력 검증과 실패 복구

- 빈 이력서나 길이 범위를 벗어난 이력서는 검증 에러로 막힌다.
- Gemini 호출이 실패하면 사용자 입력이 보존되고 재시도 가능한 안내가 표시된다.
- 키 누락은 보안 실패가 아니라 설정 실패로 안내한다.

## `AC-05` 응답 계약 유지

- 성공 응답은 `questions`, `strategy`, `summary` 구조를 유지한다.
- 각 `questions` 항목은 `question`, `category`, `keypoints` 필드를 가진다.
- 모드 차이는 응답 구조에 노출되지 않는다.
