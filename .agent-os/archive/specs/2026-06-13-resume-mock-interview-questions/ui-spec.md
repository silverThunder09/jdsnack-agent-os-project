# 모의 면접 질문 생성 UI 명세

## 화면 흐름

1. 사용자가 이력서를 텍스트로 입력하거나 PDF/DOCX를 업로드해 분석한다.
2. (선택) 사용자가 대상 직무 또는 JD 본문을 함께 입력한다.
3. 사용자가 `모의 면접 질문 생성`을 누른다.
4. 화면은 카테고리별 예상 면접 질문 목록과 준비 요약을 표시한다.

## 배치

- 모의 면접 질문 섹션은 기존 분석/매칭 리포트 흐름 뒤에 이어지는 단계로 배치한다.
- 질문은 카드 목록으로 표시하고, 각 카드는 질문 본문, 카테고리 배지, 답변 키포인트를 보여준다.
- 답변 전략과 준비 요약은 질문 목록 위 또는 아래에 요약 영역으로 표시한다.

## 성공 상태

- 생성 성공 시 질문 목록이 카테고리(`experience`/`technical`/`behavioral`) 구분과 함께 표시된다.
- 준비 요약은 핵심 준비 행동을 짧게 보여준다.

## 실패 상태

- 입력 검증 실패는 이유와 다음 행동 하나만 보여준다.
- 외부 호출 실패는 재시도 가능한 안내를 짧게 보여준다.
- 실패 시에도 이력서·직무·JD 입력값은 보존되어 바로 재시도할 수 있어야 한다.

## 입력 보존

- 질문 생성 실패는 이력서와 맥락 입력값을 지우지 않는다.
- 재요청 시 마지막 입력값을 그대로 사용할 수 있다.

## 접근성

- 이력서 입력, 직무/JD 입력, `모의 면접 질문 생성` CTA는 명확한 label과 button 역할을 유지한다.
- 오류 메시지는 `role="alert"` 또는 동등한 즉시 인지 가능한 영역에 표시한다.
- 질문 목록 영역은 결과 리포트와 동일한 focus/aria-live 흐름을 따른다.
