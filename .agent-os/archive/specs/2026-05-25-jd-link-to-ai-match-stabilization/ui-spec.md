# JD 링크 to AI 매칭 안정화 UI 명세

## 화면 흐름

1. 사용자가 이력서 텍스트를 입력하거나 PDF/DOCX를 업로드한다.
2. 사용자가 JD 본문을 직접 붙여넣거나 JD 링크를 입력한다.
3. JD 링크 수집 성공 시 JD textarea가 자동으로 채워진다.
4. 사용자가 `JD 비교 미리보기`를 누른다.
5. 화면은 AI 매칭 리포트를 표시한다.

## 성공 상태

- JD 링크 수집 성공 메시지는 짧게 표시한다.
- 자동 채움 후 사용자의 다음 행동은 `JD 비교 미리보기`다.
- 리포트는 점수, 핵심 요약, 강점, 보완 포인트, 개선 제안을 우선 표시한다.

## 실패 상태

- 링크 수집 실패 시 긴 설명을 보여주지 않는다.
- 실패 메시지는 실패 이유와 다음 행동 하나만 보여준다.
- 기본 문구는 `JD 본문을 직접 붙여넣어 주세요.`를 포함한다.
- 실패 상태에서도 JD textarea는 계속 편집 가능해야 한다.

## 입력 보존

- 링크 수집 실패는 기존 JD textarea 내용을 지우지 않는다.
- 링크 수집 성공은 새로 받은 `jdText`로 textarea를 갱신한다.
- 사용자가 이후 JD textarea를 수정하면 수정한 값이 매칭 요청 기준이 된다.

## 접근성

- JD textarea, JD URL input, 주요 CTA는 기존 label과 button 역할을 유지한다.
- 오류 메시지는 `role="alert"` 또는 동등한 즉시 인지 가능한 영역에 표시한다.
- 결과 리포트 영역은 기존 focus/aria-live 흐름을 유지한다.
