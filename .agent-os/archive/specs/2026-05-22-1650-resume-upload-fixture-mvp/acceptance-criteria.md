# 수용 기준

> 기능: JDSnack 1.5차 MVP — 이력서 업로드와 fixture 분석

## `AC-01` 텍스트 입력 분석

- 사용자가 텍스트로 이력서를 입력하면 시스템은 기존 길이 검증 규칙을 유지해야 한다.
- 유효한 입력이면 fixture 분석 결과를 반환해야 한다.

## `AC-02` PDF 업로드 분석

- 사용자가 PDF 파일을 업로드하면 시스템은 텍스트를 추출해야 한다.
- 추출 성공 시 fixture 분석 결과를 반환해야 한다.
- 추출 실패 시 업로드 실패 메시지를 반환해야 한다.

## `AC-03` DOCX 업로드 분석

- 사용자가 DOCX 파일을 업로드하면 시스템은 텍스트를 추출해야 한다.
- 추출 성공 시 fixture 분석 결과를 반환해야 한다.
- 추출 실패 시 업로드 실패 메시지를 반환해야 한다.

## `AC-04` 지원 파일 형식 제한

- PDF와 DOCX 외 파일 형식은 `400 UNSUPPORTED_FILE_TYPE`으로 거부해야 한다.

## `AC-05` fixture 결과 구조

- fixture 결과는 최소 `score`, `summary`, `strengths`, `improvements`를 포함해야 한다.
- 프론트는 이 구조를 결과 화면에 렌더링할 수 있어야 한다.

## `AC-06` 운영 모드 구분

- no-key 준비중 응답 모드와 fixture 분석 모드는 분리되어야 한다.
- 환경 설정 또는 프로필 기준으로 어느 모드를 사용하는지 결정할 수 있어야 한다.

## `AC-07` 계약 동기화

- 파일 업로드 API, 응답 구조, 화면 흐름이 바뀌면 `api-spec.md`, `ui-spec.md`, `traceability.md`가 함께 갱신되어야 한다.
