# 1.5차 MVP 개발 계획

> 목표: 실제 외부 AI 없이도 업로드, 추출, 결과 화면 흐름을 검증한다.

## Task 1: 문서 계약 정리

- 업로드 입력 방식과 fixture 결과 구조를 문서로 고정한다.
- 기존 1차 MVP와 충돌하지 않게 운영 모드 분리를 명시한다.

## Task 2: 파일 업로드 API

- `POST /api/diagnose/file` 계약을 정의한다.
- PDF/DOCX 형식 검증과 오류 코드를 고정한다.

## Task 3: 텍스트 추출 계층

- PDF와 DOCX 텍스트 추출 책임을 별도 서비스로 분리한다.
- 추출 실패 시 사용자 메시지를 표준화한다.

## Task 4: fixture 분석 결과 계층

- `DiagnosisProvider` 인터페이스를 도입한다.
- fixture 저장소와 결과 DTO를 정의한다.

## Task 5: 결과 화면

- 점수, 요약, 강점, 개선 포인트 카드 렌더링을 설계한다.
- 입력 모드 전환 UI를 추가한다.
