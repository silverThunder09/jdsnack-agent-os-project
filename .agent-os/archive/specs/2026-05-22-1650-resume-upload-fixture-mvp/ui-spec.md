# UI 명세서

> 프로젝트: JDSnack — 1.5차 MVP 이력서 업로드 + fixture 분석

## 1. 화면 목표

사용자가 텍스트 입력뿐 아니라 실제 파일 업로드를 통해 결과 화면까지 경험할 수 있게 한다.

## 2. 입력 방식

사용자는 아래 세 모드 중 하나를 선택할 수 있다.

- `Text`
- `PDF`
- `DOCX`

## 3. 입력 영역

### 3.1 Text 모드

- 기존 textarea 유지
- 글자 수 카운터 유지
- 검증 오류 유지

### 3.2 PDF 모드

- PDF 파일 업로드 버튼
- 선택한 파일명 표시
- 업로드 중 로딩 상태 표시

### 3.3 DOCX 모드

- DOCX 파일 업로드 버튼
- 선택한 파일명 표시
- 업로드 중 로딩 상태 표시

## 4. 결과 영역

fixture 분석 성공 시 아래 항목을 보여준다.

- 분석 점수
- 한 줄 요약
- 강점 목록
- 개선 포인트 목록

## 5. 오류 메시지

| 상황 | 메시지 |
|---|---|
| 지원하지 않는 파일 형식 | `PDF 또는 DOCX 파일만 업로드할 수 있습니다.` |
| 파일 텍스트 추출 실패 | `파일에서 텍스트를 추출하지 못했습니다. 다른 파일로 다시 시도해주세요.` |
| fixture 결과 없음 | `테스트 분석 결과를 찾지 못했습니다. 다른 샘플로 다시 시도해주세요.` |

## 6. 사용자 흐름

```text
사용자 접속
 -> 입력 모드 선택
 -> 텍스트 입력 또는 파일 업로드
 -> 진단 요청
 -> 텍스트 검증 또는 파일 추출
 -> stub 모드면 준비중 안내 반환
 -> fixture 모드면 결과 카드 반환
 -> 결과 상태 렌더링
```

## 7. 모드별 결과 상태

- `stub`: 준비중 안내를 보여준다.
- `fixture`: 점수, 요약, 강점, 개선 포인트 카드를 보여준다.
- 파일 업로드 오류와 fixture 없음 오류는 별도 안내 문구로 구분한다.
