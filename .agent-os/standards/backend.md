# 백엔드 표준

## 기본 원칙

- 계층 구조를 분리합니다.
- `Controller -> Service -> Repository` 흐름을 지킵니다.
- Controller는 요청/응답과 검증에 집중합니다.
- Service는 비즈니스 로직에 집중합니다.

## 코드 스타일

- 의미가 분명한 클래스 이름을 사용합니다.
- DTO와 엔티티를 분리합니다.
- 예외는 공통 방식으로 처리합니다.

## 학습 포인트

- 왜 Controller와 Service를 분리하는지 설명할 수 있어야 합니다.
- 트랜잭션 경계가 어디인지 말할 수 있어야 합니다.
