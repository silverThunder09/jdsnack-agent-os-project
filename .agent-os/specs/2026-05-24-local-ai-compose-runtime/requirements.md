# 로컬 AI Compose 런타임 요구사항

## `REQ-01` compose 기본 ai-local 실행

- 로컬 통합 실행은 백엔드를 `ai-local` 모드로 띄워야 한다.
- fixture 모드는 기본 통합 실행 경로가 아니라 테스트 안전망으로 유지한다.

## `REQ-02` 로컬 `.env` 기반 Gemini 설정

- compose는 루트 `.env`를 참조할 수 있어야 한다.
- `.env`는 로컬 전용이며 커밋하지 않는다.
- `.env`가 없어도 CI의 compose 문법 검증과 컨테이너 빌드가 깨지면 안 된다.

## `REQ-03` 비밀값 노출 방지

- `GEMINI_API_KEY` 값은 문서, 로그, PR 본문, 프론트 코드에 노출하지 않는다.
- 사용자가 브라우저에서 비밀 키를 입력하거나 저장하는 흐름은 만들지 않는다.
