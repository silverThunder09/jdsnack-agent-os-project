# 프론트 SaaS 셸 재설계 수용 기준

## `AC-01` 셸 레이아웃

- 화면 진입 시 좌측 사이드바, 상단 바, 본문 콘텐츠 영역이 함께 보인다.
- 본문은 히어로 → 통합 입력 → 결과 영역 순으로 단일 화면에 배치된다.
- 상단/사이드바 브랜드는 JDSnack으로 표기된다.

## `AC-02` 사이드바 노출·잠금·프로필 제거

- 사이드바에 전체 메뉴(홈·분석 내역·이력서 관리·템플릿·키워드 사전·맞춤 첨삭·요금제)와 모의 면접이 보인다.
- 홈과 모의 면접만 활성(이동 가능)이고, 나머지는 잠금 표시로 클릭/이동이 되지 않는다.
- 계정/프로필·플랜 영역이 화면 어디에도 없다.

## `AC-03` 통합 분석 실행

- 이력서와 JD를 입력하고 `분석 시작`을 누르면 진단과 매칭이 함께 실행된다.
- 페이지 이동 없이 결과 영역이 채워지고, 입력값은 보존되어 재실행할 수 있다.
- 진단/매칭 각각의 로딩·실패 상태가 표시된다.

## `AC-04` 결과 카드·상세

- 결과 요약 카드는 이력서 진단 점수와 JD 적합도(매칭 점수)만 표시한다(데이터 없는 ATS·키워드·첨삭 카드는 없음).
- 진단 상세(강점/개선)와 매칭 상세(강점/gap/제안)가 프리뷰 영역에 표시된다.
- 결과 생성 전에는 빈/안내 상태를 보여준다.

## `AC-05` 모의 면접 유지

- 모의 면접 목적지에서 직무(선택적으로 JD/이력서 맥락) 입력 후 질문을 생성할 수 있다.
- 질문 목록(카테고리·키포인트)·전략·요약이 표시된다.
- 모의 면접 동작은 기존과 동일하다.

## `AC-06` 계약·검증·에러 보존

- 진단/매칭/면접/JD수집 API 요청·응답 구조가 변경되지 않는다.
- 빈/짧은/긴 입력 검증, 키 누락·Gemini 실패 에러 처리가 기존과 동일하게 동작한다.
- `fixture` 모드에서 외부 호출 없이 결과가 정상 표시된다.

## `AC-07` 반응형·접근성

- 데스크톱에서 사이드바|본문, 본문 내 입력|결과의 넓은 레이아웃으로 배치된다.
- 좁은 폭에서 세로 스택으로 전환되고 사이드바가 접힘/축소된다.
- 주요 입력·버튼·결과의 label/role/포커스/오류 알림이 유지된다.
