# 프론트 SaaS 셸 재설계 UI 명세

## 전체 레이아웃 (데스크톱)

```text
┌───────────┬──────────────────────────────────────────────────┐
│ 사이드바  │  상단 바: JDSnack                       (도움말)  │
│           ├──────────────────────────────────────────────────┤
│ ▣ 홈      │  [히어로] AI 기반 이력서 분석·최적화              │
│ ▸ 모의면접 │   "공고와 이력서를 연결하고, 합격 가능성을 높이세요"│
│ ─────────  │                                                  │
│ 🔒 분석내역│  [통합 입력] 채용공고(JD 링크/붙여넣기) + 이력서  │
│ 🔒 이력서  │             (Text/PDF/DOCX)      [ 분석 시작 ]    │
│ 🔒 템플릿  │                                                  │
│ 🔒 키워드  │  [요약 카드]  진단 점수 │ JD 적합도               │
│ 🔒 첨삭    │                                                  │
│ 🔒 요금제  │  [상세 프리뷰] 진단(강점/개선) · 매칭(강점/gap/제안)│
└───────────┴──────────────────────────────────────────────────┘
```

- 계정/프로필·플랜 영역 없음(인증 없음).

## 사이드바

- 브랜드(JDSnack) + 메뉴 목록.
- 활성: `홈`, `모의 면접`(이동 가능, 현재 위치 강조).
- 잠금: `분석 내역`·`이력서 관리`·`템플릿`·`키워드 사전`·`맞춤 첨삭`·`요금제` — 잠금 아이콘, 비활성, 클릭 불가(`aria-disabled`).
- 좁은 폭: 접힘(아이콘만) 또는 상단 토글로 축소.

## 상단 바

- 좌측 브랜드 식별만 필수. 도움말 등 보조 요소는 정적(비동작) 허용.
- `STEP n / 3` 등 단계 라벨 없음.

## 홈(`/`) 본문

- 히어로: 태그·헤드라인·서브카피.
- 통합 입력: JD(링크/붙여넣기 탭) + 이력서(Text/PDF/DOCX 탭) + `분석 시작`(진단+매칭 동시). 글자 수·파일 안내 유지. 버튼은 입력 준비 상태에 따라 활성/로딩.
- 요약 카드: `이력서 진단 점수`, `JD 적합도`(매칭 점수)만. 아이콘·큰 숫자·상태 배지·프로그레스 스타일(목업 차용). 데이터 없으면 빈/안내.
- 상세 프리뷰: 진단(강점/개선)·매칭(강점/gap/제안) 패널 또는 탭. 한줄 코멘트는 `summary` 매핑.
- 상태: 빈/로딩/성공/실패를 각 영역에서 처리. 실패 시 에러+다음 행동, 입력 보존.

## 모의 면접(`/interview`) 본문

- 대상 직무(+선택 JD/이력서 맥락) 입력 + `면접 질문 생성`.
- 결과: 질문 카드(카테고리·키포인트)·전략·요약.

## 스타일

- 디자인 랭귀지: 인디고/바이올렛 강조, 라이트 배경, 둥근 흰 카드 + 소프트 섀도, 아이콘 배지, 프로그레스 바, 상태 배지(보통/양호/확인 필요 류).
- `App.css`를 셸 그리드(사이드바|본문) + 카드 시스템으로 재작성. 위저드 잔재 없음.
- 아이콘: `lucide-react` 또는 인라인 SVG(구현 시 택1).

## 반응형

- 데스크톱: 사이드바|본문 2단, 본문 내 입력|결과 넓은 배치.
- 좁은 폭: 세로 스택, 사이드바 접힘/축소, 카드 전체폭.

## 접근성

- 입력은 label 연결, 실행은 button 역할 유지.
- 오류는 `role="alert"` 등 즉시 인지 영역에 표시.
- 결과 생성 시 aria-live로 보조기기에 알림.
- 잠금 메뉴는 `aria-disabled`로 표기, 포커스 순서 자연 유지.

## 마이그레이션 메모

- 기존 입력 컴포넌트(`ResumeInput`/`ResumeFileInput`/`ResumeModeTabs`/`JdInputFields`/`DiagnoseButton`/`StatusMessage`)와 훅(`useDiagnose`/`useMatchPreview`/`useInterviewPreview`), `services/api.ts`, `types/diagnosis.ts`는 재사용한다.
- `App.tsx`/`App.css`/`AppShell.tsx`를 셸 기준으로 재작성한다(기능 동작 유지).
- 라우팅은 `react-router-dom` 도입(`/`, `/interview`) 권장. 미도입 시 뷰 상태 전환으로 대체 가능.
