# 요구사항

> 프로젝트: JDSnack — JD 입력 MVP
> 상태: 구현 진행
> 기준 결정: JD는 텍스트 붙여넣기를 우선하고, JD 링크는 선택 입력으로만 받는다.

## 사용자 문제

1.5차 MVP까지는 이력서 입력과 결과 화면은 검증되었지만, 사용자가 어떤 JD를 기준으로 비교하고 싶은지는 아직 시스템에 전달할 수 없다.

## 기능 목표

사용자는 이력서 입력 또는 업로드 이후 JD 텍스트를 직접 붙여넣고, 필요하면 JD 링크를 메모용으로 함께 남길 수 있어야 한다.

## 요구사항

### `REQ-01` JD 텍스트 입력

- 사용자는 JD 본문을 텍스트로 직접 입력할 수 있어야 한다.
- JD 텍스트는 이후 비교 분석 단계로 넘길 수 있는 기본 입력이어야 한다.

### `REQ-02` JD 링크 선택 입력

- 사용자는 JD 링크를 선택적으로 함께 입력할 수 있어야 한다.
- JD 링크는 출처 메모용이며, 이 단계에서 서버가 링크 본문을 자동 수집하지 않는다.

### `REQ-03` JD 입력 검증

- 시스템은 빈 JD, 너무 짧은 JD, 너무 긴 JD를 구분해 검증해야 한다.
- JD 링크가 있다면 URL 형식을 검증해야 한다.

### `REQ-04` 비교 API 응답 구조 제공

- 시스템은 이력서-JD 비교 분석 단계에서 사용할 요청 형태를 유지해야 한다.
- 시스템은 AI 연동 전 단계에서도 미리보기용 응답 구조를 제공해야 한다.

### `REQ-05` 화면 흐름 연결

- 사용자는 이력서 입력 또는 업로드 다음 단계에서 JD 입력 화면으로 이동할 수 있어야 한다.
- JD 입력 완료 후 다음 단계에서 비교 분석이 이어질 수 있다는 흐름을 확인할 수 있어야 한다.

## 범위 포함

- JD 텍스트 입력
- JD 링크 선택 입력
- JD 입력 검증 규칙
- 비교 API 요청/응답 구조 문서화
- JD 입력 UI 흐름 문서화
- 규칙 기반 JD 미리보기 결과 제공

## 범위 제외

- JD 링크 크롤링
- 채용 공고 HTML 본문 추출
- 실제 AI 비교 분석
- JD 저장소 영속화 설계 확장
