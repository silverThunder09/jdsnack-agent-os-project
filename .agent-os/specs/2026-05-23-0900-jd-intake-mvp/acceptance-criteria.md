# 수용 기준

> 기능: JDSnack — JD 입력 MVP

## `AC-01` JD 텍스트 입력 가능

- 사용자는 JD 텍스트를 직접 붙여넣을 수 있어야 한다.
- JD 텍스트는 필수 입력으로 취급해야 한다.

## `AC-02` JD 링크 선택 입력 가능

- 사용자는 JD 링크를 선택적으로 입력할 수 있어야 한다.
- JD 링크가 없어도 흐름은 진행 가능해야 한다.

## `AC-03` JD 텍스트 검증

- JD 텍스트가 비어 있으면 `400 EMPTY_JD`로 처리해야 한다.
- JD 텍스트가 너무 짧으면 `400 JD_TEXT_TOO_SHORT`로 처리해야 한다.
- JD 텍스트가 너무 길면 `400 JD_TEXT_TOO_LONG`으로 처리해야 한다.

## `AC-04` JD 링크 형식 검증

- JD 링크가 있을 때 URL 형식이 잘못되면 `400 INVALID_JD_URL`로 처리해야 한다.

## `AC-05` 비교 API 요청 계약 고정

- `POST /api/match/preview` 요청 초안에 `resumeSource`, `jdText`, `jdUrl`이 포함되어야 한다.
- `resumeSource.type`은 `TEXT` 또는 `FILE`만 허용하는 것으로 문서화해야 한다.
- `resumeSource.value`는 비교에 사용할 이력서 텍스트 본문을 담는 것으로 문서화해야 한다.

## `AC-06` 실제 비교 결과는 제외

- 이번 단계 문서에는 실제 점수 계산 또는 AI 비교 결과 생성을 포함하지 않아야 한다.
- 문서에는 향후 응답 필드 후보만 예약해야 한다.
- 유효한 요청은 실제 결과 대신 준비중 응답으로 종료될 수 있다.

## `AC-07` 화면 흐름 연결

- JD 입력 화면은 이력서 단계 다음 순서로 배치되어야 한다.
- 사용자는 JD 입력 완료 후 비교 분석 단계로 이어질 수 있다는 점을 이해할 수 있어야 한다.
