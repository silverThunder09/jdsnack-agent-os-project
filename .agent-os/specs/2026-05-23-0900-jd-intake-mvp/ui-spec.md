# UI 명세서

> 프로젝트: JDSnack — JD 입력 MVP

## 1. 화면 목표

사용자가 이력서 단계 다음에 JD를 직접 입력해 비교 분석 흐름의 다음 단계로 자연스럽게 넘어갈 수 있게 한다.

## 2. 입력 필드

### 2.1 JD 텍스트

- `textarea` 기반 입력
- 필수 입력
- 붙여넣기 중심 사용을 기본으로 한다

### 2.2 JD 링크

- 단일 URL 입력 필드
- 선택 입력
- 안내 문구: `채용 공고 링크는 출처 메모용입니다. 현재는 링크 본문을 자동으로 읽지 않습니다.`

## 3. 기본 상태

- 초기 상태: JD 텍스트 비어 있음, JD 링크 비어 있음
- 입력 중 상태: 글자 수 또는 입력 유효성 힌트 표시 가능
- 검증 오류 상태: JD 텍스트 오류와 JD 링크 오류를 구분해 보여준다

## 4. 오류 메시지

| 상황 | 메시지 |
|---|---|
| JD 텍스트 없음 | `JD 내용을 입력해주세요.` |
| JD 텍스트 너무 짧음 | `JD 내용이 너무 짧습니다. 핵심 자격요건이 드러나도록 더 입력해주세요.` |
| JD 텍스트 너무 김 | `JD 내용이 너무 깁니다. 핵심 본문만 정리해서 입력해주세요.` |
| JD 링크 형식 오류 | `올바른 JD 링크 형식을 입력해주세요.` |

## 5. 사용자 흐름

```text
이력서 입력 또는 업로드 완료
 -> JD 입력 단계 진입
 -> JD 텍스트 입력
 -> 필요 시 JD 링크 입력
 -> 검증
 -> 비교 분석 단계로 이동 준비
```

## 6. 범위 밖

- JD 링크 미리보기
- JD 링크 자동 파싱
- 실제 매칭 점수 렌더링
- AI 비교 결과 카드
