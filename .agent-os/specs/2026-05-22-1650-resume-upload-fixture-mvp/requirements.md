# 요구사항

> 프로젝트: JDSnack — 1.5차 MVP 이력서 업로드 + fixture 분석
> 상태: 설계
> 기준 결정: 실제 외부 AI 호출 없이, 텍스트/PDF/DOCX 입력과 테스트 DB 기반 분석 결과 화면을 먼저 검증한다.

## 사용자 문제

1차 MVP는 입력 검증과 준비중 안내까지는 확인할 수 있지만, 사용자는 실제 이력서 업로드와 분석 결과 화면 흐름을 체험할 수 없다.

## 기능 목표

사용자는 이력서를 텍스트로 입력하거나 PDF/DOCX 파일로 업로드할 수 있어야 하며, 시스템은 추출된 텍스트를 기준으로 fixture 분석 결과를 반환해야 한다.

## 요구사항

### `REQ-01` 다중 입력 방식

- 사용자는 이력서를 텍스트 입력, PDF 업로드, DOCX 업로드 중 하나로 제공할 수 있어야 한다.

### `REQ-02` 파일 텍스트 추출

- 시스템은 PDF와 DOCX 파일에서 이력서 텍스트를 추출해야 한다.
- 추출 실패 시 사용자에게 원인을 이해할 수 있는 오류를 안내해야 한다.

### `REQ-03` fixture 분석 결과 반환

- 시스템은 외부 AI 대신 테스트 DB 또는 fixture 저장소에서 준비된 분석 결과를 반환해야 한다.
- fixture 분석 결과는 점수, 요약, 강점, 개선 포인트를 포함해야 한다.

### `REQ-04` 결과 화면 검증

- 사용자는 업로드 또는 입력 후 결과 화면에서 분석 요약과 핵심 피드백을 확인할 수 있어야 한다.

### `REQ-05` 운영 모드 분리

- 시스템은 1차 MVP의 no-key 검증 모드와 1.5차 MVP의 fixture 분석 모드를 구분할 수 있어야 한다.

## 범위 포함

- 텍스트 입력
- PDF 업로드
- DOCX 업로드
- 파일 텍스트 추출
- fixture 분석 결과 반환
- 결과 화면 렌더링

## 범위 제외

- 실제 외부 AI 호출
- OCR
- 스캔 PDF 이미지 인식
- 사용자 계정 기반 파일 보관
- JD 링크 수집과 HTML 본문 추출
- 실제 채용 공고 매칭 점수 계산

## 후속 단계

- 2차 MVP에서 서버 기반 외부 AI 연동
- JD 텍스트 입력 및 JD 링크 입력
- 이력서와 JD 비교 분석
