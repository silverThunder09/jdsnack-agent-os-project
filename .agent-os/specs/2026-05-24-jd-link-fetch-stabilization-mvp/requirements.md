# JD 링크 수집 안정화 MVP 요구사항

## 기준 결정

JD 링크 수집은 `사람인` 정적 HTML 공고를 1차 지원 대상으로 한다. 서버가 JD 본문을 확신할 수 없으면 성공으로 반환하지 않는다.

## `REQ-01` 사람인 JD 본문 추출

- 사용자는 사람인 채용공고 URL을 입력해 JD 본문 초안을 가져올 수 있어야 한다.
- 서버는 HTML에서 공고 제목과 실제 직무/자격/우대사항 중심 본문을 추출해야 한다.
- `fetchMode`는 `static-html`로 반환한다.

## `REQ-02` 잘못된 성공 반환 방지

- 개인정보 안내, 채용절차 안내, 푸터, 추천공고, 광고 문구만 추출되면 성공으로 반환하지 않는다.
- 본문 품질이 기준 미달이면 실패 응답으로 처리한다.

## `REQ-03` 안전한 외부 URL 수집

- 허용 스킴은 `http`, `https`로 제한한다.
- localhost, private IP, metadata IP, 내부망 주소는 차단한다.
- timeout, redirect 제한, 응답 크기 제한을 둔다.
- 요청 헤더와 JD 원문 전체를 로그로 남기지 않는다.

## 범위

- 사람인 정적 HTML 채용공고 수집
- JD 본문 품질 기준 문서화
- 실패 코드와 테스트 시나리오 고정

## 비범위

- 잡코리아, 원티드, 리멤버 등 추가 사이트
- 로그인 필요 페이지
- JavaScript 렌더링 페이지
- anti-bot 우회
- 수집 결과 영구 저장
