# 수용 기준

> 기능: JDSnack 1차 MVP — 입력 검증과 준비중 안내

## `AC-01` 정상 입력 준비중 응답

- 50자 이상 10,000자 이하의 이력서 텍스트가 입력되면 요청 검증은 통과해야 한다.
- 검증 통과 후 실제 AI 분석 대신 `501 Not Implemented`와 `AI_ANALYSIS_NOT_ENABLED`가 반환되어야 한다.
- 사용자 메시지는 `AI 분석 기능은 준비 중입니다. 현재는 이력서 입력 검증만 가능합니다.`로 통일한다.

## `AC-02` 빈 입력 차단

- 사용자가 빈 입력으로 요청하면 시스템은 `400 EMPTY_RESUME`으로 요청을 거부해야 한다.

## `AC-03` 최소 길이 검증

- 50자 미만 텍스트는 `400 TEXT_TOO_SHORT`로 요청을 거부해야 한다.

## `AC-04` 최대 길이 검증

- 10,000자를 초과한 텍스트는 `400 TEXT_TOO_LONG`으로 요청을 거부해야 한다.

## `AC-05` 인증 정보 입력 없는 UI

- 프론트는 사용자 인증 정보 입력 모달, 설정 버튼, 저장 흐름을 제공하지 않아야 한다.

## `AC-06` UI 준비중 흐름

- 사용자는 입력, 검증, 로딩, 준비중 안내 흐름을 한 화면에서 이해할 수 있어야 한다.

## `AC-07` 계약 동기화

- API 변경 시 `api-spec.md`, UI 흐름 변경 시 `ui-spec.md`, 요구사항 변경 시 `traceability.md`가 함께 갱신되어야 한다.
