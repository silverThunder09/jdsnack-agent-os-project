# 수용 기준

> 기능: AI Resume Diagnoser MVP

## `AC-01` 정상 진단 요청

- 50자 이상 10,000자 이하의 이력서 텍스트가 입력되면 진단 요청이 성공해야 한다.
- 응답에는 `score`, `readabilityFeedback`, `contributionFeedback`, `summary`가 포함되어야 한다.

## `AC-02` 빈 입력 차단

- 사용자가 빈 입력으로 요청하면 시스템은 요청을 거부하고 안내 메시지를 반환해야 한다.

## `AC-03` 최소 길이 검증

- 50자 미만 텍스트는 요청을 거부하고 입력 길이 기준을 안내해야 한다.

## `AC-04` 최대 길이 검증

- 10,000자를 초과한 텍스트는 요청을 거부하고 최대 길이 기준을 안내해야 한다.

## `AC-05` 외부 AI 실패 처리

- Gemini API 실패 시 시스템은 내부 구현 세부사항을 노출하지 않고 재시도 가능한 메시지를 반환해야 한다.

## `AC-06` UI 결과 흐름

- 사용자는 입력, 로딩, 결과 확인 흐름을 한 화면에서 이해할 수 있어야 한다.

## `AC-07` 계약 동기화

- API 변경 시 `api-spec.md`, UI 흐름 변경 시 `ui-spec.md`, 요구사항 변경 시 `traceability.md`가 함께 갱신되어야 한다.
