# 요구사항

> 프로젝트: JDSnack — 1차 MVP 서비스 뼈대  
> 상태: 활성  
> 기준 결정: 1차 MVP에서는 사용자 인증 정보 입력과 서버 외부 AI 연동을 모두 제외한다.

## 사용자 문제

취업 준비생과 개발자는 이력서를 빠르게 점검하고 싶지만, 1차 MVP에서는 먼저 입력 화면, API 요청 흐름, 검증 기준, 준비중 안내까지 안정적으로 완성한다.

## 기능 목표

사용자가 이력서 텍스트를 입력하고 진단을 요청하면, 시스템은 입력값을 검증한 뒤 AI 분석 기능이 준비 중임을 명확하게 안내한다.

## 요구사항

### `REQ-01` 이력서 텍스트 입력

- 사용자는 이력서 텍스트를 입력하고 진단 요청을 시도할 수 있어야 한다.

### `REQ-02` 입력 검증

- 시스템은 빈 입력, 50자 미만 입력, 10,000자 초과 입력을 구분해 안내해야 한다.

### `REQ-03` AI 미연동 안내

- 유효한 입력이 들어오면 시스템은 실제 AI 분석 대신 `AI_ANALYSIS_NOT_ENABLED` 준비중 응답을 반환해야 한다.

### `REQ-04` 단순한 운영 구조

- 1차 MVP는 사용자 인증 정보 입력, 서버 비밀값 설정, 외부 AI 호출 없이 동작해야 한다.

## 범위 포함

- 이력서 텍스트 입력
- 입력 길이 검증
- `GET /api/health`
- `POST /api/diagnose` 요청/검증 흐름
- AI 분석 준비중 안내
- 프론트 입력/로딩/오류/준비중 상태 흐름

## 범위 제외

- 사용자 인증 정보 입력
- 외부 AI 실제 호출
- AI 점수 및 피드백 생성
- 회원가입/로그인
- 서버 DB 저장
- 과거 진단 히스토리 조회
- PDF/Word 직접 업로드 파싱
- JD 매칭 및 모의 면접 질문 생성

## 2차 MVP 후보

- 서버 환경변수 기반 외부 AI 연동
- `score`, `readabilityFeedback`, `contributionFeedback`, `summary` 생성
- 외부 AI 실패/타임아웃/재시도 정책
- 분석 결과 저장과 히스토리 조회
