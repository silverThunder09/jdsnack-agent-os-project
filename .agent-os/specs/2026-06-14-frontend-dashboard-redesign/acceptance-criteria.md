# 프론트 대시보드 전면 재설계 수용 기준

## `AC-01` 입력·결과 동시 표시

- 화면 진입 시 입력 영역과 결과 영역이 한 화면에 함께 보인다.
- 분석을 실행해도 입력 영역을 떠나는 페이지 전환이 일어나지 않는다.
- 단계 번호로 이동하는 위저드 UI가 존재하지 않는다.

## `AC-02` 입력 통합과 동작 유지

- 이력서(텍스트/PDF/DOCX), JD(섹션/링크), 대상 직무 입력이 한 입력 영역에 모여 있다.
- 진단/매칭/면접 질문 실행을 입력 영역에서 바로 누를 수 있다.
- 실행 후에도 입력값이 보존되어 재실행할 수 있다.

## `AC-03` 결과 패널 점진 표시

- 진단·매칭·면접 결과가 각각의 패널에 표시된다.
- 결과 생성 전 패널은 빈/안내 상태, 생성 중에는 로딩 상태를 보여준다.
- 실패 시 해당 패널에 에러와 다음 행동이 표시되고 입력값은 보존된다.

## `AC-04` 군더더기 제거 확인

- 상단에 프로필/계정 버튼이 없다.
- `STEP n / 3` 라벨이 화면 어디에도 중복 표시되지 않는다.
- 상단 바에는 서비스 브랜드만 남는다.

## `AC-05` 기존 계약·검증 보존

- 진단/매칭/면접/JD수집 API 요청·응답 구조가 변경되지 않는다.
- 빈/짧은/긴 입력 검증, 키 누락·Gemini 실패 에러 처리가 기존과 동일하게 동작한다.
- `fixture` 모드에서 외부 호출 없이 결과 패널이 정상 표시된다.

## `AC-06` 반응형 동작

- 데스크톱 폭에서 입력|결과 2단으로 배치된다.
- 좁은 폭에서 입력→결과 세로 스택으로 전환된다.
- 주요 입력·버튼·결과 영역의 label/role/포커스/오류 알림이 유지된다.
