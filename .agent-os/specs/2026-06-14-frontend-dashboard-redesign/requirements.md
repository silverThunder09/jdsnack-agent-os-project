# 프론트 대시보드 전면 재설계 요구사항

현재 3단계 위저드 화면의 문제(아래 "문제 정의")를 해소하기 위해, 입력과 결과를 한 화면에서 함께 보는 **대시보드형**으로 화면 흐름·정보구조를 전면 재설계한다. 백엔드 API와 기능 동작은 바꾸지 않는다(화면만 재구성).

## 문제 정의 (현재 화면)

- 로그인/인증 기능이 없는데 상단에 프로필 아바타 버튼이 있어 동작 없는 장식이 된다.
- 스텝 진행바와 카드 헤더에 `STEP n / 3` 라벨이 중복 표시된다.
- 스텝 진행바가 세로 공간을 과하게 차지해 입력 폼이 화면 아래로 밀린다.
- 단일 좁은 1단 컬럼이라 데스크톱에서 좌우 여백이 과하고 정보 밀도가 낮다.
- 위저드라 이력서·JD·결과를 한 번에 볼 수 없고 단계를 오가야 한다.
- 카드가 큰데 내용이 적어 화면이 비어 보인다.

## `REQ-01` 대시보드 단일 화면

- 입력 영역과 결과 영역을 한 화면에서 동시에 본다.
- 단계 전환 위저드(이력서→JD→리포트 페이지 이동)를 제거한다.
- 결과를 보기 위해 입력 화면을 떠나지 않아도 된다.

## `REQ-02` 입력 영역 통합

- 이력서 입력(텍스트/PDF/DOCX), JD 입력(섹션/링크), 대상 직무를 한 입력 영역(좌측 레일)에 모은다.
- 각 분석 실행(진단, 매칭, 면접 질문)은 입력 영역에서 바로 트리거한다.
- 입력값은 분석 후에도 보존되어 재실행할 수 있다.

## `REQ-03` 결과 영역 패널화

- 이력서 진단, JD 매칭, 모의 면접 질문 결과를 각각의 결과 패널로 표시한다.
- 패널은 결과가 생성되는 대로 채워지고, 생성 전에는 빈/안내 상태를 보여준다.
- 각 패널의 로딩·실패·입력보존 상태는 해당 패널 안에서 표시한다.

## `REQ-04` 군더더기 제거

- 인증 없는 프로필 아바타 버튼을 제거한다.
- `STEP n / 3` 라벨 중복을 제거한다(위저드 단계 개념 자체가 사라짐).
- 상단 바는 서비스 브랜드 식별만 유지한다.

## `REQ-05` 기존 기능·계약 보존

- `POST /api/diagnose`, `/diagnose/file`, `/api/jd/fetch`, `/api/match/preview`, `/api/interview/preview` 계약을 변경하지 않는다.
- 입력 검증, 에러 처리, `stub`/`fixture`/`ai-local` 모드 동작을 그대로 유지한다.
- resume source(텍스트/파일 추출 `sourceText`)를 진단·매칭·면접에 동일하게 사용한다.

## `REQ-06` 반응형 레이아웃

- 데스크톱에서는 입력|결과 2단 레이아웃을 사용한다.
- 좁은 폭(모바일/태블릿)에서는 입력→결과 세로 스택으로 전환한다.
- 접근성(label, role, 오류 알림, 포커스 흐름)은 유지한다.
