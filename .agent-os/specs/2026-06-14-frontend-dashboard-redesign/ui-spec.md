# 프론트 대시보드 전면 재설계 UI 명세

## 전체 레이아웃 (데스크톱)

```text
┌────────────────────────────────────────────────────────────┐
│  JDSnack AI                                       (상단 바)  │  ← 브랜드만, 프로필 없음
├──────────────────────────┬─────────────────────────────────┤
│  입력 레일 (좌)           │  결과 영역 (우)                  │
│                          │                                 │
│  ▸ 이력서                 │  ┌ 이력서 진단 ──────────────┐  │
│    Text / PDF / DOCX      │  │ (빈 상태 → 점수·요약·강점· │  │
│    내용 입력              │  │  개선)                     │  │
│    [이력서 진단]          │  └────────────────────────────┘  │
│                          │  ┌ JD 매칭 ──────────────────┐  │
│  ▸ JD                     │  │ (빈 상태 → 매칭점수·요약·   │  │
│    섹션 입력 / 링크 불러오기│  │  강점·gap·제안)            │  │
│    [JD 매칭]              │  └────────────────────────────┘  │
│                          │  ┌ 모의 면접 질문 ────────────┐  │
│  ▸ 대상 직무 (선택)       │  │ (빈 상태 → 질문 카드·전략·  │  │
│    [면접 질문 생성]       │  │  요약)                     │  │
│                          │  └────────────────────────────┘  │
└──────────────────────────┴─────────────────────────────────┘
```

## 입력 레일 (좌)

- 이력서/JD/대상 직무를 세로로 묶은 단일 입력 영역. 위저드 단계 이동 없음.
- 이력서: Text/PDF/DOCX 탭 + 내용 입력 + 글자 수. 실행 버튼 `이력서 진단`.
- JD: 섹션 입력과 `링크로 JD 불러오기`. 실행 버튼 `JD 매칭`.
- 대상 직무: 선택 입력. 실행 버튼 `면접 질문 생성`.
- 각 실행 버튼은 해당 입력의 준비 상태에 따라 활성/비활성. 실행 중 로딩 표시.
- 입력값은 실행 후에도 보존되어 재실행 가능.

## 결과 영역 (우)

- 진단·매칭·면접 3개의 결과 패널을 세로로 배치.
- 상태별 표시(패널 내부에서):
  - 빈 상태: "아직 결과가 없습니다 — 좌측에서 실행하세요" 류의 안내.
  - 로딩: 해당 패널만 로딩 표시.
  - 성공: 결과 내용 표시(진단=점수/요약/강점/개선, 매칭=매칭점수/요약/강점/gap/제안, 면접=질문 카드/전략/요약).
  - 실패: 에러 사유 + 다음 행동 1개, 입력은 보존.
- 결과는 생성되는 대로 독립적으로 채워진다(하나가 실패해도 다른 패널은 영향 없음).

## 상단 바

- 좌측에 서비스 브랜드(JDSnack)만. 프로필/계정/로그인 요소 없음.
- `STEP n / 3` 등 단계 라벨 없음.

## 반응형

- 데스크톱: 입력 레일 | 결과 영역 2단(예: 좌측 고정폭 + 우측 가변, 또는 4:6 비율).
- 좁은 폭(태블릿/모바일): 입력 레일 → 결과 영역 순서로 세로 스택.
- 결과 패널은 좁은 폭에서 카드 전체폭.

## 접근성

- 각 입력은 label과 연결, 실행은 button 역할 유지.
- 오류 메시지는 `role="alert"` 또는 동등한 즉시 인지 영역에 표시.
- 결과 패널은 생성 시 aria-live로 보조기기에 알린다.
- 키보드 포커스 순서는 입력 레일 → 결과 영역 자연 순서.

## 마이그레이션 메모

- 기존 `StepProgress`(위저드)와 단계 전환 상태(`currentStep`)는 제거 대상.
- 기존 `ResumeStep`/`JdStep`/`ReportStep`의 입력·결과 표시 로직은 대시보드 입력 레일/결과 패널로 재배치(기능 동작은 유지).
- 결과 표시 컴포넌트(`ResultPanel` 등)는 재사용 가능하면 재사용.
