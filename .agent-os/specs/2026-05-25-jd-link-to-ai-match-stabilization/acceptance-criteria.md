# JD 링크 to AI 매칭 안정화 수용 기준

## `AC-01` 링크 수집 성공 후 자동 채움

- 사용자가 JD 링크를 입력하고 `링크로 JD 불러오기`를 누르면 `POST /api/jd/fetch`가 호출된다.
- 성공 응답의 `data.jdText`가 JD textarea에 자동으로 들어간다.
- 성공 상태는 짧게 표시되고 사용자는 바로 `JD 비교 미리보기`를 누를 수 있다.

## `AC-02` 자동 채움 후 매칭 리포트

- 자동 채움된 JD 본문과 준비된 이력서로 `POST /api/match/preview`가 호출된다.
- 성공 시 매칭 점수, 핵심 요약, 강점, 보완 포인트, 개선 제안이 표시된다.
- 기존 매칭 결과 응답 구조와 프론트 타입은 유지된다.

## `AC-03` 링크 실패 후 복구 가능

- `POST /api/jd/fetch` 실패 시 기존 JD textarea 값은 보존된다.
- 화면은 `JD 본문을 직접 붙여넣어 주세요.`를 포함한 복구 행동을 보여준다.
- 실패 후 사용자가 JD 본문을 직접 입력하면 `JD 비교 미리보기`가 정상 동작한다.

## `AC-04` 이력서 소스별 매칭 연결

- 텍스트 이력서 모드에서는 현재 textarea 값을 매칭 요청의 resume source로 사용한다.
- PDF/DOCX 모드에서는 먼저 이력서 분석이 성공해야 매칭 요청이 가능하다.
- PDF/DOCX 분석 성공 후에는 추출된 `sourceText`로 JD 매칭을 요청할 수 있다.

## `AC-05` 범위 제외 준수

- 새 API, 새 외부 사이트 수집, 브라우저 렌더링 수집, 로그인 우회, anti-bot 우회를 추가하지 않는다.
- API 계약 변경이 필요하면 이 spec 범위 밖 변경 요청으로 남긴다.
