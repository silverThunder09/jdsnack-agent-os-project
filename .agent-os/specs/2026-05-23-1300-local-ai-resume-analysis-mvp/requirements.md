# 요구사항

> 프로젝트: JDSnack — 2차 MVP 로컬 전용 AI 이력서 분석
> 상태: 구현 대상

## 사용자 문제

현재는 이력서 진단이 `stub` 또는 `fixture` 결과에 머물러 있어, 실제 AI 분석 품질과 결과 흐름을 로컬에서 검증하기 어렵다.

## 기능 목표

사용자는 로컬 개발 환경에서 `POST /api/diagnose`와 `POST /api/diagnose/file`를 통해 실제 Gemini 기반 이력서 분석 결과를 확인할 수 있어야 한다.

## 요구사항

### `REQ-01` 로컬 AI 분석 모드

- 시스템은 `ai-local` 진단 모드를 지원해야 한다.
- `ai-local` 모드는 서버 환경변수 기반 Gemini 호출을 사용해야 한다.

### `REQ-02` 텍스트 입력 AI 분석

- 사용자는 이력서 텍스트 입력 후 실제 AI 분석 결과를 받아야 한다.
- 응답은 점수, 요약, 강점, 개선 포인트를 포함해야 한다.

### `REQ-03` 파일 업로드 AI 분석

- 사용자는 PDF/DOCX 업로드 후 텍스트 추출을 거쳐 실제 AI 분석 결과를 받아야 한다.

### `REQ-04` 구조화된 Gemini 응답

- 서버는 Gemini에 구조화된 JSON 결과를 요구해야 한다.
- 응답 파싱 실패와 호출 실패를 명확한 에러 코드로 구분해야 한다.

### `REQ-05` 사용자 키 입력 금지

- 프론트는 사용자 API Key 입력 UI를 제공하지 않는다.
- `.env`는 로컬 전용 비밀값으로 유지하고, 앱은 환경변수 존재만 사용한다.
