# 모의 면접 질문 생성 테스트 시나리오

## `TC-01` ai-local 면접 질문 생성 성공

- 대응 AC: `AC-01`
- 절차:
  - `JDSNACK_DIAGNOSIS_MODE=ai-local`로 실행하고 사용자가 로컬 `.env`에 `GEMINI_API_KEY`를 설정한다.
  - 준비된 이력서로 면접 질문 생성을 요청한다.
- 기대 결과:
  - `POST /api/interview/preview`가 호출된다.
  - 예상 면접 질문 목록과 준비 요약이 표시된다.

## `TC-02` 질문 카테고리·키포인트 표시

- 대응 AC: `AC-01`
- 절차:
  - `TC-01`의 성공 응답을 확인한다.
- 기대 결과:
  - 각 질문이 `experience`, `technical`, `behavioral` 중 하나의 카테고리를 가진다.
  - 각 질문에 답변 키포인트가 함께 표시된다.

## `TC-03` 직무·JD 맥락 반영

- 대응 AC: `AC-02`
- 절차:
  - 이력서와 함께 `jobTitle` 또는 `jdText`를 제출해 질문 생성을 요청한다.
  - 같은 이력서를 맥락 값 없이도 요청한다.
- 기대 결과:
  - 맥락 값이 있으면 질문이 해당 직무/공고에 맞춰 구체화된다.
  - 맥락 값이 없어도 요청이 정상 처리된다.

## `TC-04` stub 모드 준비중 안내

- 대응 AC: `AC-03`
- 절차:
  - `JDSNACK_DIAGNOSIS_MODE=stub`으로 실행한다.
  - 면접 질문 생성을 요청한다.
- 기대 결과:
  - 준비중 안내가 반환된다.
  - 외부 Gemini 호출이 발생하지 않는다.

## `TC-05` fixture 모드 정적 샘플

- 대응 AC: `AC-03`
- 절차:
  - `JDSNACK_DIAGNOSIS_MODE=fixture`로 실행한다.
  - 면접 질문 생성을 요청한다.
- 기대 결과:
  - 외부 호출 없이 정적 샘플 질문이 반환된다.
  - 응답 구조가 `ai-local` 성공 응답과 동일하다.

## `TC-06` 이력서 입력 검증 에러

- 대응 AC: `AC-04`
- 절차:
  - 빈 이력서로 요청한다.
  - 길이 범위를 벗어난 짧은 이력서로 요청한다.
- 기대 결과:
  - 빈 이력서는 입력 안내 에러로 막힌다.
  - 너무 짧은 이력서는 길이 안내 에러로 막힌다.

## `TC-07` Gemini 실패 시 입력 보존

- 대응 AC: `AC-04`
- 절차:
  - `ai-local` 모드에서 Gemini 호출이 실패하는 상황을 만든다.
  - 면접 질문 생성을 요청한다.
- 기대 결과:
  - 재시도 가능한 실패 안내가 표시된다.
  - 사용자가 입력한 이력서와 맥락 값이 지워지지 않는다.

## `TC-08` 응답 스키마 계약 검증

- 대응 AC: `AC-05`
- 절차:
  - 성공 응답 본문을 확인한다.
- 기대 결과:
  - 응답이 `questions`, `strategy`, `summary` 구조를 만족한다.
  - 각 `questions` 항목이 `question`, `category`, `keypoints` 필드를 가진다.
