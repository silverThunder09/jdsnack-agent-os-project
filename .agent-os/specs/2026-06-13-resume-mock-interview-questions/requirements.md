# 모의 면접 질문 생성 요구사항

## `REQ-01` 이력서 기반 면접 질문 생성

- 사용자는 준비된 이력서로 예상 면접 질문 목록을 생성할 수 있어야 한다.
- 이력서 소스는 텍스트 입력과 PDF/DOCX 분석 결과(`sourceText`)를 모두 지원한다.
- 생성 요청은 `POST /api/interview/preview` 한 번으로 질문 목록과 준비 요약을 반환한다.

## `REQ-02` 질문 분류와 답변 키포인트

- 각 질문은 `experience`, `technical`, `behavioral` 중 하나의 카테고리를 가진다.
- 각 질문은 답변에 담을 핵심 포인트(`keypoints`)를 함께 제공한다.
- 응답은 질문 목록(`questions`), 답변 전략(`strategy`), 준비 요약(`summary`) 구조를 유지한다.

## `REQ-03` 직무·JD 맥락 선택 반영

- 사용자는 대상 직무(`jobTitle`)와 JD 본문(`jdText`)을 선택적으로 함께 제출할 수 있다.
- `jobTitle` 또는 `jdText`가 있으면 질문이 해당 맥락에 맞춰 구체화돼야 한다.
- 두 값이 모두 없으면 이력서만으로 일반 예상 질문을 생성한다.

## `REQ-04` 운영 모드 분리

- `ai-local` 모드는 서버 환경변수 기반 Gemini 호출로 실제 질문을 생성한다.
- `fixture` 모드는 외부 호출 없이 정적 샘플 질문을 반환해 화면을 안전하게 검증한다.
- `stub` 모드는 준비중 안내를 반환하고 외부 호출을 하지 않는다.
- 모드는 기존 `JDSNACK_DIAGNOSIS_MODE` 설정을 재사용하며 API 계약에 모드 차이를 노출하지 않는다.

## `REQ-05` 입력 검증과 실패 복구

- 빈 이력서, 너무 짧거나 긴 이력서는 명확한 검증 에러로 막는다.
- Gemini 호출 실패 시 사용자 입력은 보존되고 재시도 가능한 에러 안내를 보여준다.
- 검증 에러와 외부 호출 실패는 기존 `ApiResponse` 실패 계약과 동일한 형식으로 반환한다.
