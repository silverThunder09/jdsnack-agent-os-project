# ATS 점수·포맷 진단 UI 계약

## 입력

- 분석 옵션의 ATS 카드는 활성 상태이며 `준비중` 태그를 표시하지 않는다.
- 기존 이력서 업로드·텍스트와 JD 링크·붙여넣기 입력을 재사용한다.

## 결과

- ATS 요약 카드에 `ATS 점수`와 0~100 점수를 표시한다.
- ATS 패널에 요약, 포맷 점검, 강점, 위험·누락, 개선 제안을 표시한다.
- 포맷 점검은 추출된 텍스트에서 확인할 수 있는 신호만 설명한다.
- 결과가 로딩·실패하면 기존 `AnalysisPanel` 상태 표현을 사용한다.
- ATS 선택이 해제되면 ATS API와 패널을 실행하지 않는다.

## 내보내기

- 성공한 ATS 결과는 기존 Markdown 내보내기에 `ATS 점수·포맷 진단` 섹션으로 포함한다.

## 접근성·보안

- 결과 패널은 기존 `aria-live` 상태 표현을 재사용한다.
- 브라우저에 API key·provider secret을 입력하거나 저장하지 않는다.
