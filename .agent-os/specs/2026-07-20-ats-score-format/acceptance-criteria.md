# ATS 점수·포맷 진단 수용 기준

- **AC-01** 인증된 사용자가 ATS 옵션을 선택하면 입력 화면에서 `준비중` 표시 없이 분석을 시작할 수 있다. (REQ-01)
- **AC-02** 유효한 이력서와 JD로 `POST /api/ats/preview`를 호출하면 `0~100` 점수와 요약·강점·위험·제안·키워드·포맷 점검을 포함한 성공 응답을 반환한다. (REQ-02, REQ-03, REQ-04)
- **AC-03** 동일한 입력을 두 번 보내면 점수와 목록 결과가 동일하다. (REQ-03)
- **AC-04** 이력서 또는 JD가 비어 있거나 50자 미만·10,000자 초과이면 기존 공통 오류 코드를 반환한다. (REQ-02)
- **AC-05** ATS 결과 화면은 점수 카드, 포맷 점검, 매칭·누락 키워드, 개선 제안을 표시한다. (REQ-04)
- **AC-06** ATS 결과가 성공하면 기존 Markdown 내보내기에 ATS 섹션이 포함된다. (REQ-04)
- **AC-07** ATS를 선택하지 않으면 ATS API를 호출하지 않고 기존 옵션 결과가 동일하게 표시된다. (REQ-05)
- **AC-08** ATS API는 외부 AI나 브라우저 secret 없이 fixture/default 실행 환경에서도 동작한다. (REQ-06)
- **AC-09** 비로그인 ATS 요청은 `401 AUTHENTICATION_REQUIRED`로 종료되고 서비스 계층을 실행하지 않는다. (REQ-02, REQ-06)
