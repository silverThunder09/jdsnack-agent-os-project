# 분석 실행 전 통합 검증 게이트 테스트 시나리오

## `TC-01` 통합 검증: 모두 통과 시 활성화

- 대응 AC: `AC-01`, `AC-02`
- 절차: 지원 형식 이력서 파일 선택 + JD 50자 이상 입력 + 분석 항목 1개 이상 선택.
- 기대 결과: 통합 검증이 통과로 판정되고 분석 시작 버튼이 활성화되며 인라인 사유가 사라진다.

## `TC-02` 짧은 JD → 버튼 비활성 + 사유

- 대응 AC: `AC-01`, `AC-02`
- 절차: 이력서 파일은 정상 선택, JD를 50자 미만으로 입력.
- 기대 결과: 분석 시작 버튼이 비활성이고 "JD를 50자 이상 입력해 주세요" 류의 인라인 사유가 표시된다.

## `TC-03` 이력서 누락 / 분석 항목 미선택 → 비활성

- 대응 AC: `AC-01`, `AC-02`
- 절차: (a) 이력서 파일 미선택, (b) 분석 항목 0개 각각 재현.
- 기대 결과: 각 경우 버튼이 비활성이고 해당 사유("이력서 파일을 업로드해 주세요", "분석 항목을 1개 이상 선택해 주세요")가 인라인 표시된다.

## `TC-04` 통과 전 호출·업로드 차단

- 대응 AC: `AC-03`
- 절차: 짧은 JD(50자 미만) + 정상 파일 상태에서 분석 시작을 시도하고, 네트워크 요청을 관찰한다(playwright 라우트/요청 가로채기).
- 기대 결과: `POST /api/diagnose/file`·`POST /api/diagnose`·`POST /api/match/preview` 요청이 발생하지 않는다(파일 업로드/추출이 먼저 일어나지 않음).

## `TC-05` 백엔드 방어적 재검증: 짧은 JD

- 대응 AC: `AC-04`
- 절차: 게이트를 우회해 JD 50자 미만으로 `POST /api/match/preview` 직접 호출(MockMvc/통합 테스트).
- 기대 결과: `400`과 `JD_TEXT_TOO_SHORT`를 `ApiResponse` 에러 형태로 반환한다.

## `TC-06` 백엔드 방어적 재검증: 짧은 이력서 텍스트/추출 본문

- 대응 AC: `AC-04`
- 절차: (a) `POST /api/diagnose`에 50자 미만 `resumeText`, (b) `POST /api/diagnose/file`에 추출 본문이 50자 미만이 되는 파일을 호출(MockMvc/통합 테스트).
- 기대 결과: 각각 `400`과 `TEXT_TOO_SHORT`(빈 값이면 `EMPTY_RESUME`)를 반환한다.

## `TC-07` 기존 흐름·게이트 종합 + 검증 게이트

- 대응 AC: `AC-05`
- 절차: 정상 입력으로 진단→매칭 실행을 확인하고, `cd frontend && npm run lint`, `npm test`, `npm run build`, `npm run test:e2e`, `cd backend && ./gradlew test`를 실행한다.
- 기대 결과: 진단→매칭 순서·결과 표시·기존 에러 처리가 유지되고, lint·단위·빌드·e2e·백엔드 테스트가 통과한다.
