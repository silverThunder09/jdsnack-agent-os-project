# 분석 실행 전 통합 검증 게이트 요구사항

분석(진단/매칭)을 실행하기 **전에** 이력서와 JD를 한 번에 검증해, 검증을 통과하지 못한 입력으로는 **AI 호출·파일 업로드가 일어나지 않도록** 막는다. 프론트는 사용자에게 사유를 즉시 보여 주고, 백엔드는 동일 기준으로 방어적으로 재검증한다.

## 문제 정의 (현재 화면)

- 분석 시작 버튼 활성 조건이 약하다: "이력서 파일 존재 + JD 비어있지 않음 + 분석 항목 1개 이상"만 본다(`frontend/src/App.tsx` `canStart`, 약 289행). JD가 50자 미만이어도 버튼이 눌린다.
- JD 최소 길이(50자) 검증이 **매칭 요청 직전**에야 일어난다(`frontend/src/hooks/useMatchPreview.ts` `validateJdText`, 약 83·151행). 그래서 짧은 JD로 분석을 시작하면 먼저 `POST /api/diagnose/file`로 파일 업로드·추출이 일어난 **뒤에야** JD 검증에 막혀, 불필요한 업로드·추출 비용이 발생한다.
- 백엔드는 이미 진단(`DiagnoseService`, 이력서 최소 50자)·매칭(`MatchPreviewService`, JD 최소 50자)을 검증하지만, 프론트 게이팅이 약해 검증 책임이 백엔드 호출 이후로 밀린다.

## 검증 규칙 (프론트·백엔드 공통 기준)

- 이력서: 분석 가능한 본문이 존재해야 한다.
  - 텍스트 입력 모드: trim 후 길이 `>= MIN_RESUME_LENGTH`(현행 50자) 이고 `<= MAX_RESUME_LENGTH`(현행 10,000자).
  - 파일 업로드 모드: 파일이 선택되어 있고 지원 형식(PDF/DOCX)이어야 한다. **추출 본문의 최소 길이는 추출 후에만 알 수 있으므로 프론트 게이트의 통과 조건이 아니며, 백엔드가 추출 직후 방어적으로 재검증한다.**
- JD: trim 후 길이 `>= MIN_JD_LENGTH`(현행 50자) 이고 `<= MAX_JD_LENGTH`(현행 10,000자).
- 분석 항목: 1개 이상 선택.
- 임계값은 현행 프론트·백엔드 코드에 이미 동일하게 박혀 있는 값(50 / 10,000)을 정본으로 채택한다(아래 "열린 질문" 참조).

## `REQ-01` 분석 실행 전 통합 검증

- 분석을 시작하기 전에 이력서(존재 + 모드별 기준)와 JD(최소·최대 길이)를 **한 번에** 검증한다.
- 검증은 진단/매칭을 실행하기 전 단계에서 수행되며, 통과한 경우에만 분석 흐름으로 진입한다.
- 검증 항목별로 통과/실패와 사유를 산출할 수 있어야 한다(인라인 표시용).

## `REQ-02` 게이팅 (통과 전 호출·업로드 차단)

- 검증을 통과하기 전에는 분석 시작 버튼이 비활성이고, 비활성 사유를 인라인으로 표시한다.
- 검증 통과 전에는 `POST /api/diagnose`, `POST /api/diagnose/file`, `POST /api/match/preview` 호출과 이력서 파일 업로드가 **발생하지 않는다.**
- 짧은 JD처럼 프론트에서 확정적으로 거를 수 있는 경우, 파일 업로드/추출(`/api/diagnose/file`)이 먼저 일어나는 일이 없어야 한다.

## `REQ-03` 프론트·백엔드 동일 기준·방어적 재검증

- 최소/최대 길이, 빈 입력, 분석 항목 선택 등 검증 규칙을 프론트와 백엔드가 동일하게 정의한다.
- 백엔드는 프론트 게이트를 신뢰하지 않고, 진단·매칭 진입 시 동일 기준으로 재검증해 위반 시 `400 Bad Request`와 명확한 `ErrorCode`를 반환한다(기존 `ApiResponse<T>` 래퍼·에러 코드 유지).
- 파일 업로드 모드에서 추출 본문이 기준 미만이면 백엔드가 추출 직후 `TEXT_TOO_SHORT` 등으로 막는다.

## 범위 밖

- 임계값(50/10,000) 자체의 변경. 이번 spec은 게이팅 시점을 앞당기고 기준을 일원화할 뿐, 숫자를 바꾸지 않는다(변경이 필요하면 "열린 질문"으로 분리).
- 새 API 엔드포인트 추가, 응답 스키마 변경.
- 이력서 텍스트 직접 입력 UI 신설(현행 입력 방식 유지). 본 spec은 기존 입력 모드 위에서 게이트만 정리한다.
