# 새로운 분석 시작 페이지 재구성 요구사항

제공된 "새로운 분석 시작" 목업대로 프론트 입력 흐름을 3단계 입력 페이지로 재구성한다. **frontend 한정이며 백엔드 API·계약·동작은 바꾸지 않는다.** 목업의 분석 옵션 중 백엔드가 없는 항목(ATS·문장 첨삭·키워드)은 "준비중"으로 노출하고, 실제 동작은 기존 기능(매칭, 진단, 면접)에 매핑한다.

## 문제 정의 (현재 화면)

- 현재 home(통합 입력+결과 동일 화면)이 목업의 단계형 입력 흐름과 다르다.
- JD 입력 방식(링크/붙여넣기)과 분석 옵션 선택이 화면에 정리돼 있지 않다.
- 분석 진행 안내(어떻게 진행되는지)와 정확도 팁이 없다.

## `REQ-01` SaaS 셸 레이아웃

- 좌측 사이드바 + 상단 바 + 본문 콘텐츠 영역으로 구성한다.
- 본문은 제목/서브카피 + 좌(입력 단계) | 우(안내 레일) 2단으로 배치한다.
- 브랜드 식별은 유지한다(제품명 변경 없음).

## `REQ-02` 사이드바·프로필·모의 면접

- 전체 메뉴(홈·분석 내역·이력서 관리·템플릿·키워드 사전·맞춤 첨삭·요금제)와 모의 면접을 노출한다.
- 동작하는 항목(홈, 모의 면접)만 활성, 백엔드 없는 항목은 잠금(비활성, 이동 불가)으로 표시한다.
- 계정/프로필·플랜 영역은 목업대로 정적(비동작) 표시한다(실제 인증 없음).

## `REQ-03` 3단계 입력

- ① 채용 공고(JD) 입력: `JD 링크`(URL 수집)와 `JD 내용 붙여넣기`(본문 직접 입력, 글자 수 표시) 탭 제공.
- ② 이력서 업로드: 드래그앤드롭/파일 선택, 업로드 파일 정보(이름·크기)와 제거 제공. 지원 형식은 백엔드가 지원하는 PDF·DOCX.
- ③ 분석 옵션 선택: 분석 항목을 체크박스로 선택한다.

## `REQ-04` 분석 옵션 동작 매핑

- 옵션 라벨은 목업대로 `JD 적합도`·`ATS 분석`·`문장 첨삭`·`키워드 분석`을 노출한다.
- `JD 적합도`만 실제 동작(매칭)하고, 나머지는 선택 시 "준비중" 안내로 처리한다.
- 옵션 선택 결과에 따라 실행되는 분석이 결정된다.

## `REQ-05` 분석 실행과 결과 화면

- `분석 시작하기`는 JD(링크/본문)·이력서 파일·옵션을 검증한 뒤 분석을 실행한다.
- `JD 적합도` 실행은 업로드 이력서에서 본문을 추출해 JD와 매칭한 결과를 만든다.
- 실행 후 결과 화면(같은 셸 내 뷰 전환)으로 이동해 결과와 "준비중" 항목을 표시한다.

## `REQ-06` 기존 기능·계약·검증 보존

- `POST /api/diagnose`, `/diagnose/file`, `/api/jd/fetch`, `/api/match/preview`, `/api/interview/preview` 계약을 변경하지 않는다.
- 입력 검증, 에러 처리, `stub`/`fixture`/`ai-local` 모드 동작을 그대로 유지한다.
- 모의 면접 기능을 보존한다.

## `REQ-07` 반응형·안내 레일·접근성

- 우측 안내 레일(진행 방식·정확도 팁·지원 형식)을 제공한다.
- 데스크톱은 입력|안내 2단, 좁은 폭은 세로 스택으로 전환한다.
- label/role/오류 알림/포커스 흐름을 유지한다.
