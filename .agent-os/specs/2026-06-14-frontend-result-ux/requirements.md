# 결과 내보내기·입력 자동저장 요구사항

"새로운 분석 시작" 페이지의 사용성을 높이기 위해 **결과 내보내기**와 **입력 자동저장·복원**을 추가한다. **frontend 한정이며 백엔드 API·계약은 바꾸지 않는다.** 새 기능은 기존 JD 적합도(매칭) 결과와 입력 흐름에 얹는다.

## 문제 정의 (현재 화면)

- 분석 결과를 따로 보관하거나 공유하려면 화면을 직접 옮겨 적어야 한다(내보내기 수단 없음).
- 새로고침하면 입력한 JD 본문과 선택한 분석 옵션이 사라진다(입력 보존 없음).

## `REQ-01` 결과 내보내기

- 결과 화면에서 JD 적합도 결과(점수·요약·강점·gap·제안)를 파일로 내려받을 수 있다.
- 내려받는 형식은 사람이 읽을 수 있는 텍스트(마크다운)다.
- 인쇄(브라우저 인쇄)로도 결과를 출력할 수 있다.
- 결과가 없을 때(분석 전/실패)는 내보내기 동작을 제공하지 않는다.

## `REQ-02` 입력 자동저장·복원

- JD 본문과 선택한 분석 옵션을 로컬에 저장하고, 새로고침 후 자동 복원한다.
- 이력서 파일은 보안·기술 제약상 저장·복원 대상에서 제외한다.
- 저장된 입력을 한 번에 지우는(초기화) 수단을 제공한다.

## `REQ-03` 기존 기능·계약·검증·접근성 보존

- `POST /api/diagnose(/file)`, `/api/jd/fetch`, `/api/match/preview`, `/api/interview/preview` 계약을 변경하지 않는다.
- 입력 검증, 에러 처리, `stub`/`fixture`/`ai-local` 모드 동작과 모의 면접 흐름을 그대로 유지한다.
- 추가 버튼·요소는 label/role/포커스 등 접근성 기준을 따르고, 반응형 레이아웃을 깨지 않는다.
