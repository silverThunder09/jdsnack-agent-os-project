# 결과 내보내기·입력 자동저장 API 명세

## 변경 없음

이번 spec은 **프론트 전용 UX 기능**(결과 내보내기, 입력 자동저장)이며 새 API를 추가하거나 기존 계약을 바꾸지 않는다.

- 내보내기/인쇄: 현재 화면에 이미 있는 매칭 결과 데이터를 클라이언트에서 파일/인쇄로 가공할 뿐, 서버 호출이 없다.
- 입력 자동저장: 브라우저 `localStorage`만 사용한다(서버 전송·저장 없음).

## 계약·보안 원칙

- `diagnose`/`match`/`interview`/`jd/fetch` 요청·응답·에러 코드·`ApiResponse<T>` 래퍼 불변.
- 이력서 파일·추출 본문은 자동저장 대상에서 제외한다(민감정보 로컬 영속화 금지).
- 비밀 키 등 민감정보를 프론트에 저장하지 않는다.
