# 결과 내보내기·입력 자동저장 UI 명세

## 결과 내보내기 (결과 화면)

- 결과 화면 헤더(또는 결과 카드 영역)에 **내보내기**·**인쇄** 버튼을 둔다. 기존 "새 분석" 버튼 옆에 배치.
- 노출 조건: JD 적합도 결과가 **성공(success)** 상태일 때만. 분석 전/로딩/실패에는 숨김(또는 비활성).
- **내보내기**: 매칭 결과를 마크다운 문자열로 만들어 `Blob` + 임시 `a[download]`로 `.md` 파일 저장.
  - 파일 내용 예: 제목, `JD 적합도 점수: N점`, 요약, 강점 목록, gap 목록, 제안 목록.
  - 파일명: `jdsnack-분석결과-YYYYMMDD.md` 형식.
- **인쇄**: `window.print()` 호출. (별도 인쇄 전용 스타일은 선택사항 — 최소한 현재 화면 인쇄.)

## 입력 자동저장·복원 (입력 화면)

- 저장 대상: JD 본문(`jdText`), 분석 옵션 선택 상태(`options`). **이력서 파일은 제외.**
- 저장 시점: 값 변경 시 `localStorage`에 저장(키 예: `jdsnack.analysis-input`).
- 복원 시점: 앱 최초 로드 시 저장값으로 초기 상태 구성. 복원되면 입력 영역에 "이전 입력을 불러왔습니다" 류의 가벼운 안내(선택).
- **입력 초기화** 버튼: JD 본문·옵션을 기본값으로 되돌리고 `localStorage` 키 삭제.
- 이력서 파일은 복원 불가하므로, 복원 후에도 파일 영역은 빈 상태(재업로드 안내 유지).

## 접근성·반응형

- 내보내기·인쇄·초기화 버튼은 명확한 텍스트 label과 button role.
- 좁은 폭에서도 버튼이 줄바꿈되며 겹치지 않게 배치.

## 마이그레이션 메모

- 결과 데이터는 기존 `useMatchPreview`의 `result.matchPreview`를 그대로 사용(추가 fetch 없음).
- 자동저장은 `App.tsx`의 `jdText`/`options` 상태에 `localStorage` 읽기(초기값)·쓰기(effect)를 추가.
- 기존 결과 표시 컴포넌트(`SummaryCard`/`AnalysisPanel`)와 셸은 변경 없이 버튼만 추가.
