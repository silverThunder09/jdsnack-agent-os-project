# 결과 내보내기·입력 자동저장 테스트 시나리오

## `TC-01` 결과 마크다운 내보내기

- 대응 AC: `AC-01`
- 절차: JD·이력서 입력 후 분석 실행 → 결과 화면에서 "내보내기" 클릭.
- 기대 결과: 점수·요약·강점·gap·제안이 담긴 `.md` 파일 다운로드가 트리거된다.

## `TC-02` 결과 인쇄

- 대응 AC: `AC-01`
- 절차: 결과 화면에서 "인쇄" 클릭.
- 기대 결과: 브라우저 인쇄(window.print)가 호출된다.

## `TC-03` 입력 자동 복원

- 대응 AC: `AC-02`
- 절차: JD 본문과 분석 옵션을 바꾼 뒤 페이지를 다시 로드한다.
- 기대 결과: JD 본문과 옵션 선택 상태가 복원된다.

## `TC-04` 이력서 파일 제외

- 대응 AC: `AC-02`
- 절차: 이력서 업로드 후 새로고침한다.
- 기대 결과: JD·옵션은 복원되지만 이력서 파일은 복원되지 않는다(재업로드 안내/빈 상태).

## `TC-05` 입력 초기화

- 대응 AC: `AC-02`
- 절차: 저장된 입력이 있는 상태에서 "입력 초기화"를 클릭한다.
- 기대 결과: JD 본문·옵션이 비워지고 로컬 저장도 삭제된다.

## `TC-06` 계약·검증 보존 및 게이트

- 대응 AC: `AC-03`
- 절차: 빈/짧은 입력 검증과 분석 흐름을 확인하고, `cd frontend && npm run lint`, `npm test`, `npm run build`, `npm run test:e2e`를 실행한다.
- 기대 결과: 기존 검증·에러 동작이 유지되고 lint·단위·빌드·e2e가 통과한다.
