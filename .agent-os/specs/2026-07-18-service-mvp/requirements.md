# Service MVP 요구사항

## 목표

로그인한 사용자가 이미 보유한 이력서와 JD를 제출해 기존 AI 진단·JD 매칭을 실행하고, 결과를 분석 이력으로 다시 확인·재시도·삭제할 수 있게 합니다.

JDSnack은 이력서를 작성하거나 별도 라이브러리로 관리하지 않습니다. 사용자가 제출한 입력은 분석 이력의 스냅샷으로만 보존합니다.

## 범위

### 포함

- Google OAuth 2.0/OIDC 1개 공급자 로그인
- 기존 이력서 텍스트 입력 또는 PDF/DOCX 업로드 후 텍스트 추출
- JD 직접 입력 또는 사람인 링크 수집
- 기존 AI 이력서 진단과 JD 매칭 연결
- 분석 입력 스냅샷·JD·결과의 사용자 소유 저장
- 성공·실패 분석 이력 목록/상세 조회
- 동일 스냅샷을 사용하는 새 분석 이력 생성 방식의 재시도
- 분석 이력과 입력·결과의 즉시 영구 삭제

### 제외

- 별도 이력서 작성기·이력서 라이브러리·이력서 버전 관리
- PDF/DOCX 원본 파일의 영속 보관
- 두 번째 OAuth 공급자와 계정 병합
- JD 최신화·주기적 재수집
- 분석 결과 export, 결제, ATS, 비동기 worker, Redis 선제 도입

## 요구사항

### REQ-01 인증과 사용자 식별

Service MVP는 Google OAuth 2.0/OIDC로 사용자를 인증하고, provider subject와 내부 사용자 식별자를 분리합니다. OAuth client secret과 provider token은 서버에서만 취급합니다. 로그인 전에는 공개 홈과 로그인 CTA만 사용할 수 있으며, 보호 API는 내부 세션이 있는 인증 사용자만 호출할 수 있습니다.

### REQ-02 기존 이력서 입력

사용자는 기존 이력서 텍스트를 직접 입력하거나 PDF/DOCX를 제출할 수 있습니다. PDF/DOCX는 텍스트 추출에만 사용하고 원본 파일은 저장하지 않습니다.

### REQ-03 JD 입력

사용자는 JD 텍스트를 직접 입력하거나 사람인 링크를 제출할 수 있습니다. 사람인 링크는 기존 JD fetch 경계를 사용하며, 분석 시점의 본문 스냅샷과 출처 URL을 저장합니다.

### REQ-04 분석 실행과 결과 저장

서버는 인증된 사용자의 입력으로 기존 이력서 진단과 JD 매칭을 실행하고, 제출 당시 이력서 텍스트·JD 본문·분석 결과를 하나의 사용자 소유 Analysis History로 저장합니다.

### REQ-05 분석 상태

분석은 최소 `RUNNING`, `SUCCEEDED`, `FAILED` 상태를 가집니다. AI 분석이 실패해도 제출 입력을 포함한 `FAILED` 이력을 저장하고 사용자에게 재시도 경로를 제공합니다.

### REQ-06 분석 이력 조회

사용자는 자신의 분석 이력만 목록과 상세로 조회할 수 있습니다. 다른 사용자의 ID를 추측해도 존재 여부가 노출되지 않아야 합니다.

### REQ-07 재시도

성공·실패 이력은 제출 당시 보존된 이력서 텍스트와 JD 스냅샷으로 재시도할 수 있습니다. 재시도는 기존 결과를 덮어쓰지 않고 새 Analysis History를 생성합니다. JD URL을 재수집하지 않습니다.

### REQ-08 삭제

사용자가 분석 이력을 삭제하면 해당 이력, 입력 사본, JD, 분석 결과를 즉시 함께 영구 삭제합니다. 삭제 후 조회·재시도·복구는 불가합니다.

### REQ-09 데이터 최소 보관

별도 이력서 라이브러리와 원본 업로드 파일은 보관하지 않습니다. 분석 이력 삭제 시 관련 입력·결과 사본도 함께 삭제합니다.

### REQ-10 계층과 계약

백엔드는 `Controller -> Service -> Repository/External API` 경계를 유지하고 Entity를 API 응답으로 직접 반환하지 않습니다. 프론트 컴포넌트는 직접 API를 호출하지 않고 서비스 계층을 사용합니다.

### REQ-11 오류와 권한

인증되지 않은 요청, 입력 검증 실패, 외부 AI/JD 수집 실패, 존재하지 않거나 타인 소유인 이력 요청을 구분된 공통 오류 계약으로 반환합니다. 타인 소유 이력은 리소스 존재 여부가 드러나지 않는 응답을 사용합니다.
