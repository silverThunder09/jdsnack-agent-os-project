# Service MVP 수용 기준

- **AC-01** 사용자는 Google 로그인 후 인증된 사용자로 식별되고, provider secret은 브라우저에 노출되지 않는다. (REQ-01)
- **AC-01a** 비로그인 사용자는 공개 홈과 로그인 CTA만 사용할 수 있고, 보호 API 요청은 `401 AUTHENTICATION_REQUIRED`로 종료된다. 로그인 전 분석·JD 수집 기능은 API를 호출하지 않는다. (REQ-01, REQ-11)
- **AC-02** 텍스트 입력 또는 PDF/DOCX 업로드로 얻은 이력서 텍스트가 분석 요청에 사용되며, 원본 파일은 저장되지 않는다. (REQ-02, REQ-09)
- **AC-03** JD 텍스트와 사람인 링크 입력이 모두 지원되고, 링크 입력은 기존 JD 수집 계약을 사용한다. (REQ-03)
- **AC-04** 성공한 분석은 입력 스냅샷·JD·진단 결과·매칭 결과·상태를 사용자 소유 이력으로 저장한다. (REQ-04)
- **AC-05** AI 실패 또는 결과 검증 실패 시에도 입력 스냅샷을 포함한 `FAILED` 이력이 저장되고 실패 사유와 재시도 경로가 반환된다. (REQ-05)
- **AC-06** 목록과 상세 조회는 현재 사용자 소유 이력만 반환하며 타인 이력의 존재 여부를 노출하지 않는다. (REQ-06, REQ-11)
- **AC-07** 성공·실패 이력 재시도는 저장된 이력서 텍스트와 JD 본문을 재사용하고, URL을 재수집하지 않으며, 새 이력 ID를 발급한다. (REQ-07)
- **AC-08** 기존 이력은 재시도로 변경되지 않고 원래 결과·생성 시각·입력 스냅샷을 유지한다. (REQ-07)
- **AC-09** 삭제 성공 후 목록·상세·재시도에서 해당 이력이 사라지고 입력·JD·결과도 함께 조회할 수 없다. (REQ-08)
- **AC-10** 이력서 라이브러리·파일 원본 보관·JD 최신화 기능이 UI와 API에 추가되지 않는다. (REQ-09)
- **AC-11** 백엔드/프론트 계층 규칙과 공통 오류 계약을 지키며 API 테스트와 UI 회귀 테스트가 통과한다. (REQ-10, REQ-11)
