# AI 품질 평가와 prompt/model version UI 계약

## T1: Prompt/Model Version

- UI 변경 없음. 모델명·프롬프트 버전은 내부 메타데이터이며 어떤 화면에도 노출하지 않는다(REQ-02).

## T2: 사용자 품질 피드백

### 위치

`frontend/src/features/analysis/AnalysisHistoryView.tsx` 상세 화면. 기존 결과(진단/매칭) 표시 영역 아래에 피드백 위젯을 추가한다.

### 구성 요소

- 좋아요(`LIKE`)/별로(`DISLIKE`) 두 개의 토글형 버튼.
- 선택적 코멘트 입력창(최대 500자, 클라이언트 측 글자수 카운터).
- 제출 버튼과 저장 완료 상태 표시(예: "피드백이 저장되었습니다").

### 상태별 동작

- `SUCCEEDED` 이력: 위젯을 노출한다. 기존에 남긴 피드백이 있으면 해당 `rating`을 선택된 상태로, `comment`를 입력창에 미리 채워 보여준다.
- `RUNNING`, `FAILED` 이력: 위젯을 노출하지 않거나 비활성화하고, 완료된 결과가 없어 평가할 수 없다는 안내를 보여준다(기존 REQ-05 상태별 내보내기 정책과 동일한 패턴, `.agent-os/archive/specs/2026-07-21-analysis-report-export/requirements.md` REQ-05 참고).
- 제출 중 오류(`409`, `400`, `401`)는 기존 오류 토스트/배너 패턴을 재사용해 사용자에게 알린다.

### 재시도와의 관계

- 재시도로 생성된 새 이력 상세 화면은 피드백 위젯이 비어있는 초기 상태로 표시된다(원본 이력의 피드백을 상속하지 않음, REQ-07).
