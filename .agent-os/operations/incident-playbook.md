# 장애 대응 플레이북

## 목적

문제가 터졌을 때 “어디부터 볼지”를 빠르게 정리하기 위한 문서입니다.

## 우선순위

1. 사용자 영향 확인
2. 헬스체크 확인
3. 최근 변경 문서와 활성 spec 확인
4. 로그 및 외부 API 상태 확인
5. 임시 완화책 적용

## 대표 장애 유형

### Gemini API 실패

- API Key 설정 확인
- 할당량/응답 지연 여부 확인
- 사용자에게 재시도 가능한 메시지 제공

### 배포 후 화면 미노출

- 정적 리소스 포함 여부 확인
- SPA 대체 라우팅 설정 확인
- 브라우저 콘솔과 네트워크 확인

### 계약 불일치

- `api-spec.md`와 실제 응답 비교
- `ui-spec.md`와 실제 화면 흐름 비교
- `traceability.md`에서 관련 요구사항과 테스트 확인
