# 장애 대응 플레이북

## 목적

문제가 터졌을 때 “어디부터 볼지”를 빠르게 정리하기 위한 문서입니다.

## 우선순위

1. 사용자 영향 확인
2. 헬스체크 확인
3. 최근 변경 문서와 활성 spec 확인
4. 로그 및 API 응답 상태 확인
5. 임시 완화책 적용

## 대표 장애 유형

### 외부 AI 연동 실패

- 1차 MVP에는 외부 AI 연동이 없으므로 이 항목은 2차 MVP부터 적용합니다.
- 2차 MVP에서 외부 AI 상태, 응답 지연, 재시도 정책을 확인합니다.
- 사용자에게 재시도 가능한 메시지를 제공합니다.

### 배포 후 화면 미노출

- 정적 리소스 포함 여부 확인
- SPA 대체 라우팅 설정 확인
- 브라우저 콘솔과 네트워크 확인

### 계약 불일치

- `api-spec.md`와 실제 응답 비교
- `ui-spec.md`와 실제 화면 흐름 비교
- `traceability.md`에서 관련 요구사항과 테스트 확인
